# Phase 9 — UI Design Contract

> Visual and interaction contract for Phase 9: Quests and Achievements.
> Generated by gsd-ui-researcher. Consumed by gsd-planner, gsd-executor, gsd-ui-auditor.

This is a **terminal-only Rich CLI project** (Python + Typer + Rich). No web framework,
no shadcn, no Tailwind. All design tokens expressed as Rich style strings and terminal
character measurements, not CSS. Spacing is measured in terminal rows/columns, not pixels.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (terminal Rich rendering) |
| Preset | not applicable |
| Component library | Rich 14.3.3 (Panels, Tables, Text, Progress, Columns) |
| Icon library | Unicode block characters (filled/empty bars, tier badges) |
| Font | Terminal monospace (inherits user's terminal font) |

Source: CLAUDE.md tech stack, confirmed by codebase scan.

---

## Spacing Scale

Terminal UI uses rows and columns, not pixels. Adapted to 8-point spirit via
character units consistent with existing render modules.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 2 chars / 1 row | Icon gaps, inline padding ("  " prefix standard in existing render) |
| sm | 4 chars / 1 row | Compact element spacing within panels |
| md | 2 rows | Default separation between panels |
| lg | 1 panel border + 1 blank row | Section separation |
| xl | Full-width panel expand=True | Major layout sections |

**Indent standard:** All panel body lines use 2-space left indent (`"  "`) — established
by `render/shop.py`, `render/battle.py`, `commands/status.py`. Phase 9 must match.

**Separator line:** `"  " + "─" * 38` for sub-section dividers within panels —
established by `render/shop.py`. Reuse this exact pattern for category dividers in
achievement and quest panels.

Exceptions: none for this phase.

---

## Typography

Terminal text uses Rich style strings. Mapped to the established "neon" default theme
from `render/themes.py`. All sizes are monospace — no font size variation.

| Role | Rich Style | Usage |
|------|-----------|-------|
| Body | `white` (weight: normal) | Quest names, achievement names, stat values |
| Label / dim | `dim white` | Category headers, progress counts, sub-labels |
| Heading | `bold white` | Panel headings, section titles |
| Accent label | `dim cyan` (theme["stat_key"]) | Field labels ("Progress:", "Reward:", "Tier:") |
| Highlight | `bold magenta` or `bold cyan` | Unlocked achievements, completed quests, notifications |

**Rule:** Never use more than 2 font weights per surface (normal + bold). Established
by all existing render modules.

Line height: terminal single-line rows. Blank row between logical groups via `"\n"`.

## Color

Terminal colors mapped to Rich style strings. Follows the neon theme default
(cyan primary, magenta accent) established in `render/themes.py`.

| Role | Rich Style | Usage |
|------|-----------|-------|
| Dominant (60%) | terminal default background | All panel backgrounds — never set explicitly |
| Secondary (30%) | `theme["border"]` = `"cyan"` (neon) / `"yellow"` (classic) | All Panel borders via `box.ROUNDED` |
| Accent (10%) | `"bold magenta"` | Reserved for: quest completion notification border, achievement unlock notification border, daily bonus payout highlight, Gold tier badge |
| Destructive | not applicable | No destructive actions in Phase 9 |

**HP-bar color thresholds (carry-over from battle.py — reuse for progress bars):**

| Threshold | Style |
|-----------|-------|
| > 66% complete | `"green"` |
| > 33% complete | `"yellow"` |
| <= 33% complete | `"dim white"` (not red — progress bars differ from HP bars) |

**Rarity colors (carry-over from RARITY_COLORS in themes.py — use for quest difficulty):**

| Difficulty | Style |
|------------|-------|
| Easy | `"white"` |
| Medium | `"bright_blue"` |
| Hard | `"magenta"` |

Source: CONTEXT.md D-03 (3 difficulty tiers), themes.py RARITY_COLORS pattern.

**Achievement tier badge colors:**

| Tier | Style | Badge glyph |
|------|-------|------------|

## Surface Inventory

Six surfaces for Phase 9. Each maps to a distinct Rich rendering function in `render/quests.py`.

### Surface 1 — Quest List (`devmon quests`)

Full-width panel titled "Active Quests". One row per quest.

**Quest row format:**
```
  [1] {Quest Name}                        {difficulty badge}
      Progress: {BarColumn 20-wide}  {n}/{target}
      Reward:   {xp} XP  +{bits} Bits  {item or empty}
```

- Quest name: `"white"`
- Difficulty badge: Easy=`"white"`, Medium=`"bright_blue"`, Hard=`"magenta"`
- Progress bar: `BarColumn(bar_width=20)` using `theme["xp_bar"]` fill color
- Progress count: `"dim cyan"`
- Reward line: `"dim white"` for labels, `"white"` for values
- Category divider between Coding / Game / Special sections: `"  " + "─" * 38` in `"dim white"`
- Border: `theme["border"]` (`box.ROUNDED`)

**Completed quest slot (post-completion, pre-refresh):**
```
  [slot] (empty — refreshes tomorrow)
```
Style: `"dim white"`.

Source: CONTEXT.md D-12, D-02.

### Surface 2 — Quest Completion Notification (deferred, on next invocation)

Full-width panel. Printed before `devmon status` panels, same timing as level-up banner.

```
  Quest Complete!
  {Quest Name}
  Reward: +{xp} XP  +{bits} Bits  {item if any}
```

- Panel title: `"[bold]Quest Complete![/bold]"`
- Border style: `"bold magenta"` (accent)
- Box: `box.DOUBLE`
- expand: `True`

If multiple quests completed since last invocation, render one panel per quest
(stacked, not batched). This matches the established level-up banner pattern.

Daily bonus panel (all 5 quests complete) is a separate panel immediately after
the last quest completion panel:
```
  Daily Bonus! All quests complete.
  Bonus: +{xp} XP  +{bits} Bits
```
Border style: `"bold cyan"` (theme levelup_text color). Box: `box.DOUBLE`. expand: `True`.

Source: CONTEXT.md D-05, D-07. Pattern from `commands/status.py` render_levelup_banner.

### Surface 3 — Achievement List (`devmon achievements`)

Full-width panel titled "Achievements". Grouped by category.

Category header row (per category: Combat, Collection, Coding, Exploration):
```
  {Category Name}
  ──────────────────────────────────────
```
Category header style: `theme["stat_key"]` (`"dim cyan"`).

**Achievement row format:**
```
  {tier badges}  {Achievement Name}          {unlock status}
                 {progress description}  {n}/{target}
```

Tier badges inline: `[B]` `[S]` `[G]` — filled for unlocked tiers, dim for locked.
- Unlocked tier badge: full color (Bronze=`"yellow"`, Silver=`"bold white"`, Gold=`"bold magenta"`)
- Locked tier badge: `"dim white"`

Achievement name: `"white"` (unlocked at least one tier) or `"dim white"` (no tiers unlocked).
Progress: `"dim cyan"` label, `"white"` count value.
Unlock status: `"bold cyan"` for most recently unlocked tier; `"dim white"` for locked.

Border: `theme["border"]`, `box.ROUNDED`, expand=`True`.

Source: CONTEXT.md D-09, D-11, D-13.

### Surface 4 — Achievement Unlock Notification (deferred, on next invocation)

Triggered same way as quest completion: pending flag pattern from `state.py`.

```
  Achievement Unlocked!
  {Achievement Name}  [{tier} Tier]
  Reward: +{xp} XP  +{bits} Bits
```

- Panel title: `"[bold]Achievement Unlocked![/bold]"`
- Border style: `"bold magenta"` (accent)
- Box: `box.DOUBLE`
- expand: `True`
- Tier label style: Bronze=`"yellow"`, Silver=`"bold white"`, Gold=`"bold magenta"`

If multiple achievements unlocked: one panel per unlock, stacked.

Source: CONTEXT.md D-10, ACHV-02.

### Surface 5 — Quests Empty State

Shown when zero active quests exist (edge case: before first daily seed).

Single panel row:
```
  No active quests. Code something today to unlock your first quest.
```
Style: `"dim white"`. Border: `theme["border"]`. expand: `True`.

Source: QUST-01, QUST-06. Default.

### Surface 6 — Achievements Empty State

Not applicable — all 20 achievements are visible from the start with 0/n progress
per CONTEXT.md D-11. There is no empty state for achievements.

## Interaction Contract

### `devmon quests`

1. Load save state
2. Process any pending event backlog (quest progress already current per D-06)
3. Check for pending quest completion notifications — render Surface 2 panels first
4. Check for pending daily bonus — render bonus panel if flagged
5. Render Surface 1 quest list (always rendered)
6. Save state (clear pending notification flags)

No interactive prompts. Read-only display command.

### `devmon achievements`

1. Load save state
2. Check for pending achievement unlock notifications — render Surface 4 panels first
3. Render Surface 3 achievement list grouped by category
4. Save state (clear pending notification flags)

No interactive prompts. Read-only display command.

### Quest progress update hook (in `progression.py process_events`)

Quest progress updates atomically with XP/stats updates — no separate user interaction.
Game-linked quest updates (battle wins, captures) triggered from battle/capture commands,
not from shell event processing.

### Notification ordering (on any `devmon` invocation)

Order of deferred notifications when multiple are pending:
1. Level-up banner (existing, from status.py)
2. Quest completion panels (one per completed quest, oldest first)
3. Daily bonus panel (if all 5 completed in same session)
4. Achievement unlock panels (one per tier unlocked, oldest first)

Then the invoked command renders normally beneath.
