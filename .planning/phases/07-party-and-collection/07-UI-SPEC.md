# Phase 7 — UI Design Contract

> Visual and interaction contract for the party-and-collection phase.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
> This is a Python + Rich terminal UI — all rendering is done via Rich panels,
> tables, progress bars, and styled text. No web framework, no shadcn, no CSS.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Rich terminal renderer) |
| Preset | not applicable |
| Component library | Rich 14.3.3 (Panel, Table, Text, Progress, Columns, box.ROUNDED) |
| Icon library | ASCII symbols only — no unicode emoji (Windows cp1252 safety) |
| Font | Terminal monospace — no custom font; Rich inherits terminal font |

**Registry safety gate:** Not applicable. No third-party component registry.

**shadcn gate result:** Not applicable. This is a Python CLI app using Rich,
not a React/Next.js/Vite app.

## Spacing Scale

Terminal spacing is expressed in blank lines and padding characters, not pixels.
The table below maps the standard 8-point scale concepts to terminal equivalents.

| Token | Terminal Equivalent | Usage |
|-------|--------------------|--------------------------------------------|
| xs | 0 blank lines, 2-char indent | Inline label-value pairs in Text blocks |
| sm | 1 blank line | Between stat rows within a panel body |
| md | 2 blank lines | Between distinct panels in a sequence |
| lg | Panel padding=1 (default) | Rich Panel internal horizontal padding |
| xl | Columns gap=4 | Horizontal gap between side-by-side panels |
| 2xl | Section heading blank line above + below | Major screen transitions |
| 3xl | n/a for MVP | Not used in Phase 7 |

Standard indent: 2 spaces for all console.print() body content. 4 spaces for
nested list items within panels.

Exceptions: Party table uses Rich Table default padding (pad_edge=False). ASCII
art blocks use no additional indent — art defines its own whitespace.

## Typography

Terminal text does not have px sizes or weights in the CSS sense. The equivalent
is Rich style strings. All styles below are Rich markup keywords.

| Role | Rich Style | Usage |
|------|-----------|-------|
| Body | `white` or `dim white` | Standard output lines, stat values, descriptions |
| Label | `dim cyan` | Stat key labels ("LVL", "HP", "Type", "Name") |
| Heading | `bold white` | Panel titles, section headings, prompt questions |
| Accent/Rarity | `RARITY_COLORS[rarity]` | Creature names, rarity badges, border colors |

**Source:** Matches existing `themes.py` + `creatures.py` + `battle.py` patterns.
No new styles introduced in Phase 7.

Line height: 1 terminal line per text row. Blank lines used for rhythm — see
Spacing Scale above.

## Color

All colors are Rich style strings from the established palette. Hex values are
not applicable in terminal rendering.

| Role | Rich Value | Usage (60/30/10 mapping) |
|------|-----------|--------------------------|
| Dominant surface (60%) | terminal default (black/dark) | Terminal background — Rich renders on top |
| Secondary surface (30%) | `dim white` / `dim cyan` | Labels, inactive items, empty slots, unseen codex entries |
| Accent (10%) | `cyan` (neon theme), `yellow` (classic) | Active panel borders, CTA prompts, progress bar fill |
| Rarity accent | per `RARITY_COLORS` — see below | Creature names, panel borders, [P] badge |
| Destructive | `bold red` | Fainted indicator, error messages only |
| Positive | `bold green` | HP bar when >50%, success confirmation lines |

**Rarity color mapping (source: `render/themes.py` — do not redefine):**

| Rarity | Rich Style |
|--------|-----------|
| common | `white` |
| uncommon | `green` |
| rare | `bright_blue` |
| epic | `magenta` |
| legendary | `bold yellow` |

**Accent reserved for:**
- Active panel border (`border_style` on ROUNDED panels)
- XP bar fill color (`xp_bar` theme key)
- Interactive prompt text ("What will you do?", swap slot prompts)
- Codex completeness progress bar fill

**Destructive color reserved for:** fainted creature indicator in party table
and party detail. Not used for routine display.

**HARD RULE:** Never display `capture_rate` anywhere in Phase 7 UI. This value
is internal to the game engine only. (Source: project feedback memory.)

## Screen Inventory

All Phase 7 views and their layout contracts.

### Screen 1: `devmon party` — Party Overview Table

**Trigger:** `devmon party` with no subcommand

**Layout:** Rich Table printed to console (not a panel). Followed by an empty
line, then a prompt if the party is below 3.

**Table spec:**
- Title: "Active Party" styled `bold` + theme `title` color
- box: `box.SIMPLE` (matches compact read-only tables used elsewhere)
- 5 columns: Slot, Name, Level, HP, Status
- No `pad_edge`; consistent 1-space column padding

**Column definitions:**

| Column | Width | Style | Source |
|--------|-------|-------|--------|
| Slot | 4 chars | `dim white` | "1", "2", "3" or "[Empty]" |
| Name | 20 chars (truncated) | RARITY_COLORS of creature rarity | OwnedCreature.nickname or template.name |
| Level | 5 chars | `white` | OwnedCreature.level |
| HP | 12 chars | HP color thresholds (see below) | "current/max" |
| Status | 10 chars | see status rules | "OK", "FAINTED", "[Empty]" |

**HP color thresholds (consistent with `render_hp_bar()`):**
- >50% HP: `green`
- 26-50% HP: `yellow`
- 0-25% HP: `red`

**Status cell rules:**
- Active, healthy: `OK` styled `dim white`
- Fainted: `FAINTED` styled `bold red`
- Empty slot: `--` styled `dim white`; Slot cell shows `[Empty]` styled `dim white`

**Party indicator [P] in collection:** When a creature appears in collection
list and is also in the active party, append ` [P]` to its name cell, styled
`dim cyan`. (Source: CONTEXT.md D-07)

**Empty party state:** When all 3 slots are empty (no creatures collected yet),
print the table with 3 empty rows, then print:
> "Your party is empty. Capture a creature in battle to get started."
styled `dim white`.

---

### Screen 2: `devmon party swap <slot> [creature_name]` — Party Swap

**Trigger:** `devmon party swap 1` (interactive) or `devmon party swap 1 Bugbyte` (direct)

**Interactive mode layout:**
1. Print current party table (same as Screen 1)
2. Print blank line
3. Print numbered list of available (non-fainted, not already in slot) creatures
   from collection — one per line: `  [N] CreatureName  LVL X  (rarity)`
   styled by rarity color
4. Print prompt: `Choose a creature [1-N, or 0 to cancel]: ` styled `bold white`
5. On invalid input: re-prompt once, then abort with error copy (see Copywriting)
6. On success: print confirmation (see Copywriting)

**Direct mode (power user):** Skip menu. Validate slot (1-3) and creature name.
Print confirmation or error. No interactive prompts.

**Fainted creature rule:** Fainted creatures must NOT appear in the swap list.
They are displayed in the collection table (greyed out) but cannot be selected
for party slots. (Source: REQUIREMENTS.md PRTY-04)

---

### Screen 3: `devmon collection` — Collection Table

**Trigger:** `devmon collection` with no subcommand

**Layout:** Rich Table followed by a codex completeness line.

**Table spec:**
- Title: "Your Collection" styled `bold` + theme `title` color
- box: `box.SIMPLE`
- Default sort: rarity (rarest first — legendary, epic, rare, uncommon, common)
- Sort flags: `--sort rarity|level|name` (source: CONTEXT.md D-06)
- 6 columns: #, Name, Rarity, Level, Type, Status

**Column definitions:**

| Column | Width | Style | Notes |
|--------|-------|-------|-------|
| # | 3 chars | `dim white` | Row index |
| Name | 22 chars | RARITY_COLORS of creature rarity | Nickname or template name; append ` [P]` dim cyan if in party |
| Rarity | 10 chars | RARITY_COLORS of creature rarity | "Common", "Uncommon", etc. |
| Level | 5 chars | `white` | OwnedCreature.level |
| Type | 10 chars | `dim white` | CreatureTemplate.type |
| Status | 10 chars | see rules | "OK" or "FAINTED" (bold red) |

**Codex line below table:**
> "Codex: 8/25 discovered"
followed immediately by a Rich Progress bar (width=20, cyan fill, dim white bg).
Styled: label `dim white`, counts `white`, bar uses theme `xp_bar` color.
(Source: CONTEXT.md D-10)

**Empty collection state:** Print table header with no rows, then:
> "No creatures captured yet. Use 'devmon battle' to start your collection."
styled `dim white`.

### Screen 4: `devmon collection <name>` — Creature Detail

**Trigger:** `devmon collection Bugbyte` (or partial name match, case-insensitive)

**Layout:** Reuse `render_creature_panel()` from `render/creatures.py` exactly.
(Source: CONTEXT.md canonical refs, CONTEXT.md D-05)

Add one additional line below the panel showing party status:
- If in party: `  Party slot: {N}` styled `dim cyan`
- If not in party: `  Not in active party.` styled `dim white`
- If fainted: `  Status: FAINTED — needs rest before next battle.` styled `bold red`

**Nickname display:** If OwnedCreature has a nickname, use it as the panel
title instead of template.name. No "(Species)" suffix. (Source: CONTEXT.md D-13)

**No-match behavior:** If name does not match any owned creature, print error
copy (see Copywriting).

---

### Screen 5: `devmon collection rename <creature> <new_name>` (or interactive)

**Trigger:** `devmon collection rename` (interactive picker) or
`devmon collection rename Bugbyte "Byte"` (direct)

**Interactive mode layout:**
1. Print numbered collection list (name + level + rarity, one per line)
2. Prompt: `Which creature to rename [1-N]: ` styled `bold white`
3. Prompt: `New name (max 20 chars): ` styled `bold white`
4. On success: print confirmation (see Copywriting)
5. On invalid name: print validation error, re-prompt once, then abort

**Direct mode:** Validate creature name match and new_name. Print confirmation
or error. No interactive prompt.

**Validation rules (source: CONTEXT.md D-12):**
- Max 20 characters
- No empty string
- No other restrictions

### Screen 6: Codex (subcommand or section)

**Design decision (Claude's discretion):** Codex is a subcommand of collection:
`devmon collection codex`. This keeps the command surface flat and consistent.

**Layout:** Rich Table, single column layout.

**Table spec:**
- Title: "Creature Codex" styled `bold` + theme `title` color
- box: `box.SIMPLE`
- 4 columns: #, Name, Rarity, Discovery
- All 25 creatures listed; sorted by ID order (canonical creature list order)

**Discovery state display (source: CONTEXT.md D-08, D-09):**

| State | Name Display | Rarity Display | Discovery Cell |
|-------|-------------|---------------|----------------|
| Unknown | `???` styled `dim white` | `???` styled `dim white` | `Unseen` styled `dim white` |
| Encountered (seen/battled) | template.name styled `dim` + RARITY_COLORS | rarity styled `dim` | `Encountered` styled `dim cyan` |
| Captured | template.name styled RARITY_COLORS (full) | rarity styled (full) | `Captured` styled `dim green` |

**Note:** "Encountered" covers all non-captured seen states (seen, battled,
defeated). CONTEXT.md D-08 specifies 3-state tracking only.

**Codex header line above table:**
> "Codex: 8/25 discovered  [progress bar]"
Same format as collection table footer.

## Copywriting Contract

All player-facing text. Exact strings for executor to use.

| Element | Copy |
|---------|------|
| Primary CTA (party swap prompt) | `Choose a creature [1-N, or 0 to cancel]:` |
| Primary CTA (rename prompt) | `New name (max 20 chars):` |
| Party empty state | `Your party is empty. Capture a creature in battle to get started.` |
| Collection empty state | `No creatures captured yet. Use 'devmon battle' to start your collection.` |
| Empty party slot cell | `[Empty]` |
| Slot below max hint | `Tip: Use 'devmon party swap <slot>' to fill your party.` |
| Party swap success | `{nickname} moved to slot {N}.` |
| Rename success | `{old_name} renamed to {new_name}.` |
| Rename validation error (empty) | `Name cannot be empty. Please enter a name.` |
| Rename validation error (too long) | `Name must be 20 characters or fewer.` |
| Creature not found error | `No creature named '{query}' in your collection.` |
| Invalid slot error | `Slot must be 1, 2, or 3.` |
| Fainted creature swap error | `{name} has fainted and cannot join the party. Use a revive item first.` |
| No valid creatures error | `No available creatures to add to this slot.` |
| Swap cancel | `Swap cancelled.` |
| Rename cancel | `Rename cancelled.` |
| Codex header | `Codex: {N}/25 discovered` |
| Party column heading | `[P]` badge appended to name in collection: ` [P]` |

**Destructive actions in Phase 7:** None. Renaming and party management are
reversible. No confirmation dialogs required.

## Interaction Contracts

### Input model
All interactive prompts use `input()` (stdlib) or `typer.prompt()`. Rich Live
must NOT be active during any interactive prompt. (Source: STATE.md decision
from Phase 6 — "Live context exited before capture animation and interactive
sub-prompts".)

### Numeric menu pattern
Numbered lists always start at 1. Option 0 is always "cancel". Invalid input
re-prompts once; second invalid input aborts with "Swap cancelled." or "Rename
cancelled." styled `dim white`.

### Partial name matching for collection detail and rename
Case-insensitive substring match on nickname first, then template name. If
multiple creatures match, list all matches and ask user to be more specific.
If zero creatures match, print "Creature not found" error and exit.

### Table max-width constraint
All tables must render within 80 columns. Column widths specified above are
set so the sum does not exceed 78 columns (leaving 2 for table borders).
If a creature name exceeds its column width, truncate with `...` at col-3.

### Fainted creature display rule
Fainted creatures appear in collection and codex tables with `FAINTED` styled
`bold red` in the Status column. They appear greyed (append `dim` to name
style) in party swap interactive lists. They do NOT appear in swap candidate
lists.
