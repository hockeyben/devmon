    1. **Party display**: Run `devmon party`
       - Verify 3-slot table with creature name, level, HP, status
       - If party has creatures: verify rarity-colored names, HP values, "OK" status
       - If empty slots: verify "[Empty]" placeholders

    2. **Party swap (direct)**: Run `devmon party swap 1 <creature_name>`
       - Verify creature moves to slot 1
       - Run `devmon party` again to confirm change persisted

       - Verify numbered candidate list appears (fainted creatures excluded)
       - Select a creature, verify confirmation message
       - Run `devmon party` to confirm

    4. **Collection table**: Run `devmon collection`
       - Verify table shows all captured creatures with rarity colors
       - Verify party members have [P] badge
       - Verify "Codex: N/25 discovered" line below table
       - Try `devmon collection --sort level` and `--sort name`

    5. **Collection detail**: Run `devmon collection <creature_name>`
       - Verify full creature panel with ASCII art, stats, flavor text
       - Verify party slot info below panel

    6. **Codex**: Run `devmon collection codex`
       - Verify all 25 creatures listed
       - Verify unknown creatures show "???" entries
       - Verify encountered/captured creatures show correct state
       - Verify progress bar at top

    7. **Rename**: Run `devmon collection rename <creature> <new_name>`
       - Verify confirmation message
       - Run `devmon party` and `devmon collection` to verify nickname appears everywhere
       - Try empty name and 21+ char name to verify validation errors

    8. **Edge cases**:
       - Verify "Slot must be 1, 2, or 3" on invalid slot
       - Verify "No creature named..." on non-existent creature
       - Verify capture_rate is NEVER displayed anywhere
