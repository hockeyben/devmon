  <name>Task 1: display_name helper and party swap command</name>
  <files>src/devmon/render/party.py, src/devmon/commands/party.py, tests/test_party.py</files>
  <action>
    1. Create `src/devmon/render/party.py` (NEW):
       - Pure render helper module. No I/O, no state mutation.
       - `display_name(owned: OwnedCreature, template: CreatureTemplate) -> str`: Return `owned.nickname if owned.nickname else template.name`. Per D-13, nicknames replace species name everywhere with no "(Species)" suffix.
       - Import this helper in party.py and update the party table rendering from Plan 01 to use it instead of inline `owned.nickname or template.name`.

    2. Add `swap_cmd` subcommand to `src/devmon/commands/party.py`:
       - `@app.command("swap")`
       - Signature: `swap_cmd(slot: int = typer.Argument(..., help="Party slot 1-3"), creature_name: Optional[str] = typer.Argument(None, help="Creature name to assign"))`
       - Validate slot is 1, 2, or 3. If not, print "Slot must be 1, 2, or 3." styled dim white and return.
       - Load state. If None, print "No save file found." and return.
       - Build candidate list: non-fainted creatures from creature_collection that are NOT already in state.party (or are in the slot being swapped). Per PRTY-04, fainted creatures excluded.
       - If no candidates: print "No available creatures to add to this slot." styled dim white and return.

       **Direct mode** (creature_name provided):
       - Case-insensitive match creature_name against candidates using display_name(). Substring match: `query.lower() in display_name(c, t).lower()`.
       - If zero matches: print "No creature named '{creature_name}' in your collection." and return.
       - If multiple matches: print list and ask user to be more specific.
       - If exactly one match: assign to slot.

       **Interactive mode** (creature_name is None):
       - Print current party table (reuse the party_cmd display logic — extract table rendering into a helper function `_render_party_table(state, console)`)
       - Print blank line
       - Print numbered list of candidates: `  [N] {display_name}  LVL {level}  ({rarity})` — styled by rarity color via RARITY_COLORS
       - Prompt: `Choose a creature [1-N, or 0 to cancel]: ` styled bold white, using `input()` (not typer.prompt, consistent with battle.py pattern)
       - On "0" or empty: print "Swap cancelled." dim white and return
       - On invalid input: re-prompt once, then print "Swap cancelled." and abort
       - On valid selection: assign creature to slot

       **Assignment logic (shared by both modes):**
       - Extend state.party list to `slot` length if needed (fill with empty strings or handle index)
       - Actually: state.party is a list of template_ids. To set slot N (1-indexed):
         - While `len(state.party) < slot`: `state.party.append("")` (placeholder)
         - `state.party[slot - 1] = selected_creature.template_id`
         - Remove any empty string placeholders from the end: `while state.party and state.party[-1] == "": state.party.pop()`
         - Enforce max 3: `state.party = state.party[:3]`
       - If the selected creature was already in a different slot, remove it from that slot (swap, not duplicate)
       - Save state via save_state()
       - Print confirmation: "{display_name} moved to slot {slot}." styled white

    3. Add tests to `tests/test_party.py`:
       - `test_party_swap_direct_mode`: Create state with creatures, invoke `party swap 1 Bugbyte` via CliRunner, assert "moved to slot 1" in output, assert state.party[0] == "bugbyte"
       - `test_party_swap_invalid_slot`: Invoke `party swap 5 Bugbyte`, assert "Slot must be 1, 2, or 3" in output
       - `test_fainted_excluded_from_swap`: Create state with one fainted creature, invoke party swap, assert fainted creature NOT in candidate list output
       - `test_party_swap_preserves_save`: After swap, reload state, assert party change persisted
       - `test_display_name_nickname`: OwnedCreature with nickname="Sparky" -> display_name returns "Sparky"
       - `test_display_name_no_nickname`: OwnedCreature with nickname=None -> display_name returns template.name
       - `test_party_swap_case_insensitive`: Invoke `party swap 1 bugbyte` (lowercase), assert it matches "Bugbyte"
