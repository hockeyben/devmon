# Phase 999.1: Improve ASCII Art for All 25 Creatures - Context

**Gathered:** 2026-04-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Upgrade all 25+ creature ASCII art from minimal geometric shapes (5-6 lines) to detailed, recognizable full-body terminal art. Also create evolution art showing before/after transformation. This phase is purely visual — no gameplay, stat, or data model changes.

</domain>

<decisions>
## Implementation Decisions

### Art Style & Detail Level
- **D-01:** Full-body detailed art — 10-15 lines tall, complete creatures with limbs, tails, features. Not just faces.
- **D-02:** Realistic proportions — anatomically plausible designs with ASCII shading, not cartoony or geometric.
- **D-03:** Pose varies by personality — aggressive creatures get action/fighting poses, gentle ones get calm/natural poses. Match creature nature.

### Consistency Rules
- **D-04:** Flexible canvas by size class — Small: 8-10 lines, Medium: 10-14 lines, Large/Legendary: 14-18 lines. Max 20 cols wide. No fixed single size.

### Color Integration
- **D-05:** Use Rich markup directly in ascii_art strings — e.g., `[bold red]` tags within the art array. Battle renderer already uses Rich, so this integrates naturally.

### Batch Approach
- **D-06:** Batch by rarity tier — Common creatures first, then uncommon, rare, legendary. Players encounter common creatures most frequently, so they get the upgrade first.

### Evolution Art
- **D-07:** Before/after display during evolution — show old creature art next to new evolved form side-by-side when evolution triggers.
- **D-08:** Evolved form size at Claude's discretion per creature — some may be bigger, some may keep the same canvas with more detail. Whatever looks best for the pair.

### Claude's Discretion
- Evolved form sizing (D-08)
- Specific ASCII art designs for each creature
- Which shading characters to use for detail/texture
- Exact line counts within the size class ranges

</decisions>
