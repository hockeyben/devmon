<objective>
Update the creature art validator and both render files to support Rich markup tags embedded in ascii_art strings, and add tests to verify the changes.

Purpose: D-05 requires Rich markup directly in art strings for per-character coloring. The current renderer treats art as plain text (Text.append with style=), causing markup to render as literal characters. The validator counts markup tag characters toward width limits, causing false rejections. Both must be fixed before any art is written with markup.

Output: Markup-aware validator, markup-capable renderers, passing test suite.
</objective>

@model_validator(mode="after")
def _validate_ascii_art(self) -> "CreatureTemplate":
    lines = self.ascii_art
    if len(lines) < 3:
        raise ValueError("ASCII art must have at least 3 lines")
    if len(lines) > 20:
        raise ValueError("ASCII art must not exceed 20 lines")
    for i, line in enumerate(lines):
        if len(line) > 40:
            raise ValueError(
                f"ASCII art line {i + 1} exceeds 40-char limit ({len(line)} chars)"
            )
    return self
```

                 f"ASCII art line {i + 1} exceeds 40-char visual limit ({len(visual_line)} visual chars)"

         ```

    2. In src/devmon/render/creatures.py (line 57-61), replace the art-building loop:
       FROM:
       ```python

       for i, line in enumerate(template.ascii_art):
           if i > 0:
               art.append("\n")
           art.append(line, style=template.primary_color)
       ```
       TO:
       ```python
       art = Text()
       for i, line in enumerate(template.ascii_art):
           if i > 0:
               art.append("\n")

       ```
       Note: This removes the uniform primary_color application. Art lines that contain no markup will render as unstyled text. This is intentional per D-05 — all new art will embed its own Rich markup for coloring. Existing plain art will still display (just without color until upgraded).
