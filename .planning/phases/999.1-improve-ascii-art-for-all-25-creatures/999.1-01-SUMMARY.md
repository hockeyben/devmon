# Phase 999.1 Plan 01: Markup-Aware Validator and Renderer Summary

**Rich markup support enabled in ascii_art strings: _MARKUP_RE validator strips tags before width measurement, both renderers use Text.from_markup(), and four creature JSONs have literal brackets escaped**

## Accomplishments

- `_MARKUP_RE` module constant added to `creature.py` — validator now strips markup tags before measuring visual width, so markup-heavy art lines pass the 40-char check correctly
- Both `render/creatures.py` and `render/battle.py` updated to use `Text.from_markup(line)` via `append_text()` — markup tags in art strings now render as styled Rich text instead of literal characters
- Four creature JSON files fixed: `boulder_bash`, `moss_golem`, `cyber_beetle`, `stackcat` had unescaped `[` characters that would cause `MarkupError` with the new renderer — all escaped as `\\[` in JSON
- Three new tests added: `test_art_markup_renders_correctly` (passes), `test_art_visual_width_strips_markup` (passes), `test_art_line_counts_by_rarity` (xfail, art upgrade pending)

## Files Created/Modified

- `src/devmon/models/creature.py` — Added `import re`, `_MARKUP_RE` constant, updated `_validate_ascii_art` to use `visual_line = _MARKUP_RE.sub("", line)`, updated `ascii_art` docstring
- `src/devmon/render/creatures.py` — Replaced `art.append(line, style=template.primary_color)` with `art.append_text(Text.from_markup(line))`
- `src/devmon/render/battle.py` — Same replacement in `render_battle_creature_panel`
- `src/devmon/data/creatures/boulder_bash.json` — Escaped `[##] [##]` as `\\[##] \\[##]`
- `src/devmon/data/creatures/moss_golem.json` — Escaped `[_] [_]` as `\\[_] \\[_]`
- `src/devmon/data/creatures/cyber_beetle.json` — Escaped `[=]` as `\\[=]`
- `src/devmon/data/creatures/stackcat.json` — Escaped `[__]` as `\\[__]`
- `tests/test_creatures.py` — Added three new test functions, updated error message regex match

## Decisions Made

- Renderers drop the uniform `primary_color` style application. Plain art lines now render without color until they are upgraded with markup. This is intentional per D-05 — all new art will embed its own Rich markup, and existing plain art still displays correctly (just unstyled).
- `_MARKUP_RE` is module-level (not inside the validator method) so tests can import it directly for unit testing the strip behavior.
- The `test_art_line_counts_by_rarity` test is marked `xfail(strict=False)` — it will correctly fail until plans 02-04 complete the art upgrade.
