<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Verify encounter system visual rendering and CLI flow</name>
  <what-built>
    Complete encounter system: encounter spawning engine, devmon encounter command with action menu, notification one-liner, PS1 paw indicator, flee/expiry messages, and encounter type visual treatments. All automated tests pass.
  </what-built>
  <how-to-verify>
    **Pre-check: Full test suite**
    Run `uv run pytest tests/ -v` and confirm all tests pass with 0 failures.

    **1. Encounter Screen (ENCR-05, CLI-09, D-19)**
    To test without waiting for a real encounter, manually queue one:
    ```bash
    uv run python -c "
    from devmon.persistence.save import load, save
    from devmon.models.encounter import EncounterEntry
    import time
    state = load()
    if state is None:
        from devmon.models.state import GameState
        state = GameState.new_game('Player')
    state.encounter_queue = EncounterEntry(
        template_id='inferno_drake',
        encounter_level=12,
        encounter_type='elite',
        rarity='epic',
        queued_at=time.time()
    )
    save(state)
    print('Encounter queued!')
    "
    ```
    Then run `uv run devmon encounter`. Verify:
    - Creature panel shows ASCII art, name, stats (LVL, HP, ATK, DEF, SPD, Type, Capture), flavor text
    - Panel border is magenta (epic rarity color)
    - Subtitle shows "Elite Encounter" in dim text
    - LVL row appears as first stat row showing "12"
    - Action menu: "[1] Battle", "[2] Flee", "[3] Items (coming soon)" — items in dim
    - Input prompt: "Enter choice [1-3]: "

    **2. Action Menu Responses (D-19, D-22)**
    - Enter "1" -> prints "Battle system coming in Phase 6! Encounter preserved."
    - Re-queue encounter, run again, enter "2" -> prints flee confirmation with creature name in rarity color, "No XP lost."
    - Re-queue encounter, run again, enter "3" -> prints "Items not available yet." then re-prompts
    - Enter "7" -> prints "Invalid choice." then re-prompts

    **3. Empty State (D-21)**
    Clear the encounter queue:
    ```bash
    uv run python -c "
    from devmon.persistence.save import load, save
    state = load()
    state.encounter_queue = None
    save(state)
    "
    ```
    Run `uv run devmon encounter`. Verify:
    - Prints: "No wild creatures nearby. Keep coding — one will appear soon!"

    **4. PS1 Indicator (D-06)**
    Queue an encounter (step 1 script), then run:
    ```bash
    uv run devmon prompt
    ```
    Verify output contains "🐾" and the `| 🐾 >` pattern.
    Clear encounter queue, run again, verify "🐾" is NOT present.

    **5. Boss Encounter Visual (D-12)**
    Queue a boss encounter:
    ```bash
    uv run python -c "
    from devmon.persistence.save import load, save
    from devmon.models.encounter import EncounterEntry
    import time
    state = load()
    state.encounter_queue = EncounterEntry(
        template_id='void_leviathan',
        encounter_level=25,
        encounter_type='boss',
        rarity='legendary',
        queued_at=time.time()
    )
    save(state)
    "
    ```
    Run `uv run devmon encounter`. Verify:
    - Panel border is bold yellow (legendary)
    - Subtitle shows "BOSS ENCOUNTER" in bold red
    - Feels dramatic and eye-catching

    **6. Notification One-Liner (D-05, UI-02)**
    Check notification formatting by running:
    ```bash
    uv run python -c "
    from devmon.engine.encounter_engine import format_encounter_notification
    print(format_encounter_notification('EmberFox', 'uncommon'))
    print(format_encounter_notification('VoidLeviathan', 'legendary'))
    "
    ```
    Verify creature names appear in their rarity colors (green for uncommon, bold yellow for legendary).

    **7. devmon --help**
    Run `uv run devmon --help` and verify "encounter" appears as a subcommand.
  </how-to-verify>
