    **Function: tick_encounter(state: GameState, config: dict, now: float | None = None) -> str | None**
    - Per D-08: if state.encounter_queue is not None, return None (one encounter at a time)
    - `now = now or time.time()`
    - Per D-01: Check normal timer: if now >= state.encounter_cooldown_until AND now >= state.last_encounter_time + ENCOUNTER_TICK_INTERVAL_SECONDS
      - Per D-02: chance = ENCOUNTER_BASE_CHANCE + (state.encounter_roll_count * ENCOUNTER_CHANCE_ESCALATION)
      - Roll random.random() < chance
      - If miss: increment state.encounter_roll_count, update state.last_encounter_time = now
      - If hit: spawn encounter (see below)
    - Per D-03: If state.ai_session_active AND separate AI timer check (now >= state.last_encounter_time + AI_BOOST_INTERVAL_SECONDS)
      - Roll random.random() < AI_BOOST_CHANCE
      - If hit: spawn encounter AND reset normal cooldown (D-03)
    - **Spawn logic:** call select_encounter_creature(load_all_creatures()), roll_encounter_type(), compute_encounter_level(). Create EncounterEntry. Set state.encounter_queue = entry. Reset state.encounter_roll_count = 0. Set state.encounter_cooldown_until = now + ENCOUNTER_COOLDOWN_SECONDS. Increment state.total_encounters_seen. Append to state.encounter_history (capped at ENCOUNTER_HISTORY_MAX — trim oldest if over). Update state.last_encounter_time = now.
    - Return format_encounter_notification(creature_name, rarity) on spawn, None otherwise.

    **Function: check_expiry(state: GameState, now: float | None = None) -> str | None**
    - Per D-09: if state.encounter_queue is not None and now > entry.queued_at + ENCOUNTER_TIMEOUT_SECONDS
      - Look up creature name via get_creature(entry.template_id).name
      - Clear state.encounter_queue = None
      - Increment state.expired_count
      - Return format_expiry_message(creature_name, entry.rarity)
    - Return None if no expiry

    **Function: process_ai_events(state: GameState, events: list[dict]) -> None**
    - Scan events for type="ai_start" — set state.ai_session_active = True
    - Note: ai_active is a session-level flag. For MVP, any ai_start event in the batch sets it True. It gets cleared on next invocation if no ai_start events are present.
    - This keeps AI detection simple: flag is set per-invocation based on recent events.
