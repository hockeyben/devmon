# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Encounter System.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase adds five distinct UI surfaces: (1) compact encounter notification
> one-liner, (2) full `devmon encounter` inspection screen, (3) PS1 indicator
> when an encounter is queued, (4) expiry/flee message one-liner, and
> (5) empty state for `devmon encounter` when no encounter is queued.
> All rendering uses Rich — no web, no GUI, no shadcn. Spacing and typography
> sections map to Rich layout concepts, not CSS.

## Spacing Scale

Rich panels and terminal layouts use character-cell units, not CSS pixels.
The 8-point pixel scale is mapped to character equivalents below.
This table is identical to Phase 4 — no new exceptions in Phase 5.

| Token | Pixel equiv | Char equiv | Usage |
|-------|-------------|------------|-------|
| xs | 4px | 1 char | Inline padding inside Text blocks |
| sm | 8px | 2 chars | Panel padding (Rich default: 1 char each side) |
| md | 16px | 4 chars | Gap between side-by-side panels in Columns |
| lg | 24px | 6 chars | Blank lines between panel groups |
| xl | 32px | 8 chars | Not used in Phase 5 |
| 2xl | 48px | 12 chars | Not used in Phase 5 |
| 3xl | 64px | 16 chars | Not used in Phase 5 |

Exceptions:
- ASCII art line width: max 40 chars per line (enforced by Phase 4 Pydantic validator —
  unchanged). Encounter screen inherits this constraint via `render_creature_panel()`.
- Action menu items: no indent. Single blank line separator between creature panel
  and action menu block.
- Touch targets: not applicable (terminal, keyboard-only).

Source: 04-UI-SPEC.md Spacing Scale.

## Typography

Terminal rendering has no font size — all text is monospace at the terminal's
native size. "Size" below maps to Rich semantic styling (bold, dim, etc.).

| Role | Rich Style | Usage |
|------|-----------|-------|
| Display | `bold {rarity_color}` | Creature name in encounter panel title |
| Heading | `bold white` | Action menu header ("What will you do?") |
| Body | `white` | Action menu items (Battle, Flee), notification text |
| Label | `dim {theme.stat_key}` | Stat key labels in creature panel ("HP", "ATK") |
| Muted | `dim white` | Panel subtitle (rarity + type), flavor text, grayed items |
| PS1 plain | plain UTF-8 string (no Rich) | PS1 prompt output via sys.stdout.buffer.write() |

Line height: 1 line per row (terminal default). No configurable line spacing.

Source: 04-UI-SPEC.md Typography, prompt.py PS1 pattern (D-07 no ANSI in PS1 output).

## Color

This project uses Rich style strings, not hex values. The 60/30/10 split maps
to semantic layer responsibilities in the terminal.

| Role | Rich Style | Usage |
|------|------------|-------|
| Dominant (60%) | terminal background (no fill) | Panel interior, notification line background |
| Secondary (30%) | `dim` overlays, panel borders via `theme["border"]` | Panel borders, stat label dimming, grayed-out Items menu item |
| Accent (10%) | rarity color per RARITY_COLORS map (see below) | Encounter panel border, creature name, notification creature name |
| Destructive | `bold red` | Not used in Phase 5 — Flee has no penalty, no destructive state |

Accent reserved for: encounter panel border color, creature name in panel title,
creature name in the compact notification one-liner, creature name in the expiry
one-liner. Never used for action menu items, stat values, flavor text, or the
empty state message.

Source: CONTEXT.md D-05 (rarity color on creature name in notification),
04-UI-SPEC.md Color, themes.py RARITY_COLORS.

| Rarity | Rich Style | Usage in Phase 5 |
|--------|-----------|-----------------|
| common | `white` | Notification name, panel border |

Source: themes.py RARITY_COLORS (verified in codebase).

### Encounter Type Visual Treatment

Encounter type (Normal / Rare / Elite / Boss) is displayed in the encounter
panel subtitle alongside rarity. No separate color — type is differentiated
by label text only.

| Encounter Type | Subtitle Treatment |
|---------------|-------------------|
| normal | Omitted from subtitle (implicit default) |
| rare | `[dim]Rare Encounter[/dim]` appended after rarity-type line |
| elite | `[dim]Elite Encounter[/dim]` appended after rarity-type line |
| boss | `[bold red]BOSS ENCOUNTER[/bold red]` appended after rarity-type line |

Rationale: Boss is the only encounter type warranting accent treatment because
it represents a genuinely alarming event. Elite uses dim to signal "stronger
but manageable." Rare encounter type (distinct from rare rarity) uses dim to
avoid conflation with the rarity accent color. Normal is silent.

Source: CONTEXT.md D-12, D-13 (encounter type frequency and bonuses).

## UI Surface Contracts

This section defines the five distinct UI surfaces introduced in Phase 5.

### Surface 1: Compact Encounter Notification One-Liner

Triggered: After a shell command fires and the encounter engine queues a new
creature. Output via `print()` / `typer.echo()` — NOT Rich Console — to avoid
interfering with the user's command output. Must be a single line.

Format:
```
⚡ A wild [{rarity_color}]{creature_name}[/{rarity_color}] appeared! Use devmon encounter to inspect.
```

Rules:
- Exactly one line. No panel, no border, no multi-line block.
- Creature name styled with its rarity color (RARITY_COLORS lookup).
- "⚡" is a literal Unicode character — no Rich markup around it.
- Message text before and after the creature name: `white` (terminal default,
  no explicit style needed).
- Output via `console.print()` with a shared Console instance. Not `typer.echo()`
  (typer.echo strips Rich markup).
- This fires once when the encounter is queued, then is silent.

Example outputs:
```
⚡ A wild Bugbyte appeared! Use devmon encounter to inspect.          # common: white
⚡ A wild EmberFox appeared! Use devmon encounter to inspect.         # uncommon: green
⚡ A wild Nullhound appeared! Use devmon encounter to inspect.        # rare: bright_blue
⚡ A wild Voidwraith appeared! Use devmon encounter to inspect.       # epic: magenta
⚡ A wild Auroracle appeared! Use devmon encounter to inspect.        # legendary: bold yellow
```

Source: CONTEXT.md D-05.

### Surface 2: Full Encounter Inspection Screen (`devmon encounter`)

Triggered: `devmon encounter` command when an encounter is queued.

Layout (top to bottom, single column):

```
{render_creature_panel() output — rarity-colored ROUNDED panel with ASCII art + stats + flavor}

  What will you do?

  [1] Battle
  [2] Flee
  [3] Items  (coming soon)

  Enter choice [1-3]:
```

Rules:
- `render_creature_panel()` from `render/creatures.py` is called directly —
  no duplicate implementation. Pass the encounter's `CreatureTemplate` + active
  Console + active theme.
- The encounter creature's level is displayed in the panel. Add a `level` field
  display row to the stat block: `Lv. {encounter_level}` shown beneath the
  creature name or as a stat row in the panel. Exact placement: new first row
  in the stat block, above HP.
  Format: `LVL ` (stat_key style) + `{encounter_level}` (stat_value style).
- Blank line between panel bottom and action menu heading.
- Action menu heading: `"  What will you do?"` — 2-space indent, `bold white`.
- Blank line between heading and first menu item.
- Menu items: 2-space indent, `white`.
  - `[1] Battle` — always active
  - `[2] Flee` — always active
  - `[3] Items  (coming soon)` — `dim white` entire line. Grayed to communicate
    it is non-functional until Phase 8. Must still be parseable (selecting [3]
    shows: "Items not available yet. Coming in a future update.").
- Blank line between last menu item and prompt.
- Input prompt: `"  Enter choice [1-3]: "` — 2-space indent, `white`. Use
  `typer.prompt()` or `input()` — NOT Rich Prompt (keeps it PS1-safe and avoids
  async).
- Panel width: `expand=False` (inherits from `render_creature_panel()`).
  Action menu is unindented relative to terminal margin (not indented to match
  panel). The 2-space indent on menu items creates visual grouping without
  requiring panel-width alignment.

Source: CONTEXT.md D-19, D-20 (encounter screen content), render/creatures.py
(render_creature_panel signature), 04-UI-SPEC.md Creature Panel Layout Contract.

### Surface 3: PS1 Prompt Indicator

Triggered: `devmon prompt` command — called on every shell prompt render.
When an encounter is queued in GameState, append `🐾 ` before the `>` terminator.

No-encounter format (current, unchanged):
```
⚡ Lv.{level} | XP: {earned}/{needed} >
```

Encounter-queued format:
```
⚡ Lv.{level} | XP: {earned}/{needed} | 🐾 >
```

Rules:
- Plain UTF-8 string — no Rich markup, no ANSI escape codes. PS1 output has
  always been plain text; this indicator follows the same constraint.
- `🐾` is a single Unicode emoji. Rendered by the user's terminal font.
- Separator before 🐾: ` | ` (space-pipe-space) matching the existing separator
  pattern between other PS1 segments.
- Disappears automatically when encounter is resolved (battled, fled, or expired)
  because `devmon prompt` re-reads GameState on every invocation.
- Implementation: conditional `if state.encounter_queue is not None` in
  `commands/prompt.py` before building the output string.

Source: CONTEXT.md D-06, prompt.py (existing PS1 pattern).

### Surface 4: Expiry / Flee Message One-Liner

Triggered: Next shell command after encounter expires (1-hour timeout), or
immediately after user selects Flee in the encounter screen.

**Expiry message** (shown after timed-out encounter is cleared):
```
The wild [{rarity_color}]{creature_name}[/{rarity_color}] got tired of waiting and fled!
```

**Flee confirmation** (shown immediately after user selects [2] Flee):
```
You fled from [{rarity_color}]{creature_name}[/{rarity_color}]. No XP lost.
```

Rules:
- Both: exactly one line. No panel, no border.
- Creature name styled with rarity color (RARITY_COLORS lookup).
- Output via `console.print()` with shared Console instance.
- Expiry fires once on next command, same delivery mechanism as notification
  (postexec hook → on startup processing → single print before returning to
  shell). One-shot flag cleared after print.
- Flee confirmation fires synchronously inside `devmon encounter` flow before
  returning to shell.

Source: CONTEXT.md D-07 (expiry copy), D-22 (flee has no XP penalty).

### Surface 5: Empty State for `devmon encounter` (No Encounter Queued)

Triggered: `devmon encounter` when `GameState.encounter_queue` is `None`.

Format: Single `console.print()` call — no panel, no border.

```
No wild creatures nearby. Keep coding — one will appear soon!
```

Rules:
- Plain `white` text. No rarity color (there is no creature).
- No panel wrapper. Inline text only.
- Exactly one line. No follow-up prompt.
- Command exits cleanly after printing (exit code 0).

Source: CONTEXT.md D-21.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Encounter notification | "⚡ A wild {creature_name} appeared! Use devmon encounter to inspect." |
| Encounter action menu heading | "What will you do?" |
| Battle action | "[1] Battle" |
| Flee action | "[2] Flee" |
| Items action (disabled) | "[3] Items  (coming soon)" |
| Items selected (disabled) | "Items not available yet. Coming in a future update." |
| Input prompt | "Enter choice [1-3]: " |
| Flee confirmation | "You fled from {creature_name}. No XP lost." |
| Expiry one-liner | "The wild {creature_name} got tired of waiting and fled!" |
| Empty state | "No wild creatures nearby. Keep coding — one will appear soon!" |
| Invalid input | "Invalid choice. Enter 1, 2, or 3." |

Tone: Friendly and game-like. Matches the Pokémon-inspired "wild X appeared!"
pattern from CONTEXT.md Specific Ideas. Flee is explicitly penalty-free (D-22)
so "No XP lost." reinforces that fleeing is a supported, non-punishing action.

Creature name formatting in copy: always use `template.name` (Title Case display
name as stored in creature JSON, e.g., "EmberFox", "Bugbyte"). Never lowercase.

Source: CONTEXT.md D-05, D-07, D-21, D-22.

Destructive actions in Phase 5: none. Flee is explicitly non-destructive (D-22).
No confirmation dialog required for any action.

## Interaction Flow Contract

### `devmon encounter` — Happy Path (Encounter Queued)

```
User runs: devmon encounter
  → load GameState
  → if encounter_queue is None: print empty state, exit 0
  → else:
      render_creature_panel(encounter.template, console, theme)
      print blank line
      print "  What will you do?"
      print blank line
      print "  [1] Battle"
      print "  [2] Flee"
      print "  [3] Items  (coming soon)"  ← dim white
      print blank line
      choice = input("  Enter choice [1-3]: ")
      if choice == "1":
          → invoke devmon battle flow (Phase 6 wires this — Phase 5 stubs out)
      elif choice == "2":
          → clear encounter_queue, save state
          → print flee confirmation one-liner
          → exit 0
      elif choice == "3":
          → print "Items not available yet. Coming in a future update."
          → re-prompt (loop back to menu)
      else:
          → print "Invalid choice. Enter 1, 2, or 3."
          → re-prompt (loop back to menu)
```

Phase 5 Battle stub: `devmon encounter` choosing [1] in Phase 5 prints
"Battle system coming in Phase 6! Encounter preserved." and exits 0 without
clearing the encounter queue. Phase 6 replaces this stub with real battle logic.

### PS1 Indicator — Lifecycle

```
Encounter spawns
  → encounter_queue set in GameState, saved
  → notification one-liner printed to shell (postexec)
  → devmon prompt now outputs 🐾 indicator on every shell prompt

User runs devmon encounter → Flee
  → encounter_queue cleared, saved
  → flee confirmation printed
  → devmon prompt no longer shows 🐾

Encounter expires (>60 min)
  → on next postexec: expiry one-liner printed, encounter_queue cleared, saved
  → devmon prompt no longer shows 🐾
```

Source: CONTEXT.md D-06, D-07, D-08, D-09.

## Encounter Level Display

The creature's encounter level is NOT stored in `CreatureTemplate` (which holds
base stats only). It is computed by the encounter engine and stored in the
encounter queue entry as `encounter_level: int`.

The encounter inspection screen must display this computed level alongside the
base stats. Implementation:

`render_creature_panel()` accepts an optional `encounter_level: int | None = None`
parameter. When provided, a `LVL` row is inserted as the first row of the stat
block (before HP). When None (roster viewer, collection viewer), the row is
omitted. This preserves backward compatibility with Phase 4 and Phase 7 usages.

Stat block with encounter level:
```
LVL  {encounter_level}    Type    {type}
HP   {base_hp}      SPD     {base_speed}
ATK  {base_attack}  Capture {capture_rate}
DEF  {base_defense}
```

Source: CONTEXT.md D-15, D-16, D-17 (encounter level formula), D-19 (screen
content includes level).
