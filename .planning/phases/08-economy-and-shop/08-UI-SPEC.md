## Spacing Scale

Rich terminal spacing uses character-cell units, not pixel values. The 8-point pixel scale is mapped to character equivalents for consistency.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 1 char | Icon gaps, inline label padding (e.g. `"HP  "` dim label) |
| sm | 2 chars | Compact stat row spacing (e.g. `"  "` between columns) |
| md | 4 chars | Default element indent (e.g. `"    "` reward lines in screens) |
| lg | 1 blank line | Section break within a panel body |
| xl | 2 blank lines | Between distinct panels or screen sections |
| 2xl | Panel border | Rich Panel separates major layout zones |
| 3xl | Full-width panel | `expand=True` for page-level separators (level-up banner) |

Exceptions:
- Reward lines inside victory/capture screens use 4-char indent (`"    "`) — matches existing Phase 6 render pattern.
- Shop category headers use 2-char indent (`"  "`) within panel body — matches action menu pattern.
- `input("  Press Enter to continue.")` uses 2-char indent — carry forward from Phase 6.

**Source:** Existing patterns in render/battle.py, commands/status.py.

## Typography

Terminal rendering has no px sizes — Rich uses semantic styles mapped to terminal escape codes. The type system is expressed as Rich style strings.

| Role | Rich Style | Usage |
|------|-----------|-------|
| Body | `"white"` | Item names, quantities, prices, descriptions |
| Label | `"dim cyan"` (neon) / `"dim white"` (classic) | Stat keys, category headers, column labels |
| Heading | `"bold white"` | Section headings within panels, action prompts |
| Accent heading | `theme["title"]` (`"bold cyan"` neon / `"bold yellow"` classic) | Panel title text |

Line height: single-line per row with explicit `\n` separators — Rich monospace default.
Weight equivalents: `"dim"` = regular (400), `"bold"` = semibold (600).

**Source:** render/themes.py semantic keys, render/battle.py, commands/status.py patterns.

## Color

| Role | Rich Style | Usage |
|------|-----------|-------|
| Dominant (60%) | terminal background (implicit) | All panel bodies, interstitial text |
| Secondary (30%) | `theme["border"]` (`"cyan"` neon / `"yellow"` classic) | Panel borders, table borders |
| Accent (10%) | `"bold green"` | Purchase success confirmation, sufficient-funds price display |
| Destructive | `"dim white"` + grayed styling | Unaffordable items grayed out; no destructive delete actions in this phase |

Accent reserved for:
- Purchase success panel border and title (green = confirmed transaction)
- Bits balance display when positive (>0 Bits)
- Item price text when player can afford the item

Rarity colors carry forward from Phase 4 (RARITY_COLORS in themes.py):
- common: `"white"`, uncommon: `"green"`, rare: `"bright_blue"`, epic: `"magenta"`, legendary: `"bold yellow"`

Shop-specific semantic colors:
- Affordable item price: `"green"`
- Unaffordable item price: `"dim white"` (grayed — per D-21)
- Bits balance in header: `"bold white"` always visible
- XP booster active indicator: `"bold magenta"` (matches theme["level"])

**Source:** CONTEXT.md D-03, D-21, D-05; themes.py RARITY_COLORS; render/battle.py color conventions.

## Screen Inventory

All surfaces rendered as Rich Panels with `box=box.ROUNDED` and `expand=False` unless noted.

### Surface 1: `devmon shop` — Shop Listing

Layout: Full-width Panel per category (Capsules, Potions, Boosters), stacked vertically.

```
╭─ Shop ─────────────────────────────────────╮
│  Bits: 120                                  │
╰─────────────────────────────────────────────╯

╭─ Capsules ──────────────────────────────────╮
│  [1] Basic Capsule        5 Bits   x0       │
│  [2] Great Capsule       12 Bits   x0       │
│  [3] Ultra Capsule       30 Bits   x0       │
│      Master Capsule   (earn only)           │
╰─────────────────────────────────────────────╯

╭─ Potions ───────────────────────────────────╮
│  [4] Small Potion         8 Bits   x3       │
│  [5] Full Potion         20 Bits   x0       │
╰─────────────────────────────────────────────╯

╭─ Boosters ──────────────────────────────────╮
│  [6] XP Booster          25 Bits   x0       │
╰─────────────────────────────────────────────╯

  Enter number to buy (or q to quit):
```

- Shop header panel: `border_style=theme["border"]`, `expand=True`
- Bits balance label: `"dim cyan"` / value: `"bold white"`
- Category panels: `border_style=theme["border"]`, `expand=True`
- Affordable price: `style="green"`
- Unaffordable price: `style="dim white"` (grayed)
- Master Capsule earn-only: `style="dim white"` with `(earn only)` suffix, no number prefix
- Quantity shown as `x{N}` in `style="dim white"` after price
- Prompt line: 2-char indent, `style="bold white"`

### Surface 2: Purchase Confirmation Panel

Shown inline after successful purchase — styled Rich Panel, not a separate screen.

```
╭─ Purchased ─────────────────────────────────╮
│  Great Capsule x1                           │
│  Cost: -12 Bits                             │
│  Balance: 108 Bits                          │
╰─────────────────────────────────────────────╯
```

- `border_style="green"`, `box=box.ROUNDED`, `expand=False`
- Title: `"[bold green]Purchased[/bold green]"`

- Balance row: `"dim white"` label + `style="bold white"` for value

## Item Catalog (Design Reference)

Claude's discretion per CONTEXT.md — established here as the design contract.

### Capsules

| Item | Effect | Price | Notes |
|------|--------|-------|-------|
| Basic Capsule | 1.0x capture multiplier | 5 Bits | Starter item |
| Great Capsule | 1.5x capture multiplier | 12 Bits | — |
| Ultra Capsule | 2.0x capture multiplier | 30 Bits | Real currency sink |
| Master Capsule | 100% capture rate | Not sold | Earn only (quests/achievements) |

### Potions

| Item | Effect | Price | Notes |
|------|--------|-------|-------|
| Small Potion | Restores 25% of max HP | 8 Bits | Usable in battle or from items menu |
| Full Potion | Restores 100% HP | 20 Bits | Real currency sink |
| Revive | Restores fainted creature to 50% HP | 15 Bits | Per D-09 |

### Boosters

| Item | Effect | Price | Notes |
|------|--------|-------|-------|
| XP Booster | 1.5x XP for 30 real minutes | 25 Bits | Per D-08; timer in game state |

**Pricing rationale (D-10):** Battle win awards ~8-15 Bits. Basic Capsule (5 Bits) = less than 1 battle. Ultra Capsule (30 Bits) = 2-4 battles. Sinks feel earned, not grindy.

| Element | Copy |
|---------|------|
| Primary CTA (shop purchase) | `"Enter number to buy (or q to quit):"` |
| Primary CTA (items use) | `"Use which item? (or b to go back):"` |
| Primary CTA (capture sub-menu) | `"Throw which capsule? (or b to go back):"` |

## Interaction Patterns

### Interactive Shop (`devmon shop`)

1. Print shop header panel (Bits balance).
2. Print one panel per category with numbered items.
3. Prompt: `input("  Enter number to buy (or q to quit): ")`.
4. On valid number: validate funds → show error or show purchase confirmation panel → re-print shop (loop).
5. On `q` or `Q`: exit with no message.
6. Invalid input: `"  Invalid choice."` in `style="dim white"`, re-prompt immediately.

### Quick Purchase (`devmon shop --buy <item-id> [--qty N]`)

1. Load state, validate item ID and quantity.
2. If insufficient funds: print error line, exit with code 1.
3. If valid: deduct Bits, add to inventory, print purchase confirmation panel, save, exit.

**Source:** CONTEXT.md D-02.

### Battle Item Sub-Menu

1. Live context paused (Rich Live cannot be active — same pattern as Phase 6 capture animation).
2. Print items sub-menu inline.
3. Prompt: `input("  Choose: ")`.
4. On valid choice with qty > 0: apply effect, print `"{ItemName} used on {CreatureName}."`, return to battle loop (turn consumed).
5. On `b`: return to action menu without consuming turn.
6. Invalid or zero-qty choice: `"  Invalid choice."` dim, re-prompt.

**Source:** CONTEXT.md D-15; Phase 6 pattern (Live exited before interactive sub-prompts).

### Battle Capture Sub-Menu

Same pattern as Battle Item Sub-Menu. On valid capsule selection: proceed to existing `run_capture_animation()`. On `b`: return to action menu without consuming turn.

**Source:** CONTEXT.md D-18.
