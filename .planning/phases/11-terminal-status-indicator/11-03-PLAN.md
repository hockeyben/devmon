**`src/devmon/shell/hooks.py`**: Update both `_devmon_preexec()` and `_devmon_precmd()` in `BASH_ZSH_HOOK_SNIPPET` to manage the typing flag file and daemon auto-start.

The typing flag mechanism (RESEARCH.md Pitfall 1 mitigation, SC6):
- `_devmon_preexec()` creates `typing.flag` — this signals "user is executing a command / readline may be active". The daemon sees this flag and skips terminal writes.
- `_devmon_precmd()` deletes `typing.flag` — this signals "prompt is being drawn, readline is not active, safe to write". The daemon sees no flag and resumes writing.

The flag file location must match `typing_flag_path()` in `indicator.py`: `${DEVMON_HOME:-$HOME/.local/share/devmon/devmon}/typing.flag`

The full updated functions become:
```bash
_devmon_preexec() {
  _DEVMON_CMD_START=$(date +%s%3N)
  # SC6: Signal daemon that readline/command is active — do not write to terminal
  touch "${DEVMON_HOME:-$HOME/.local/share/devmon/devmon}/typing.flag" 2>/dev/null
}
_devmon_precmd() {
  local _exit=$?
  # SC6: Signal daemon that prompt is drawing — safe to write to terminal
  rm -f "${DEVMON_HOME:-$HOME/.local/share/devmon/devmon}/typing.flag" 2>/dev/null
  local _now
  _now=$(date +%s%3N)
  local _dur=$(( _now - ${_DEVMON_CMD_START:-$_now} ))
  local _log="${DEVMON_EVENT_LOG:-$HOME/.local/share/devmon/devmon/events.log}"
  printf '{"ts":%s,"exit":%d,"dur":%d,"cwd":"%s","type":"cmd"}\n' \
    "$_now" "$_exit" "$_dur" "$PWD" >> "$_log" 2>/dev/null
  # Indicator daemon auto-start (Phase 11, D-02)
  local _pid_file="${DEVMON_HOME:-$HOME/.local/share/devmon/devmon}/indicator.pid"
  if [[ ! -f "$_pid_file" ]] || ! kill -0 "$(cat "$_pid_file" 2>/dev/null)" 2>/dev/null; then
    devmon indicator start >/dev/null 2>&1 &
    disown
  fi
  _DEVMON_CMD_START=
}
```

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-11-11 | Denial of Service | Stale typing.flag from shell crash | mitigate | precmd unconditionally deletes typing.flag on every prompt draw. Next terminal session starts without flag. Daemon treats missing flag as safe-to-write (default behavior). |
