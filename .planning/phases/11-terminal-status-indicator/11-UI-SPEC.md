# Phase 11 — UI Design Contract

> Visual and interaction contract for the terminal status indicator daemon.
> This is a terminal-rendered Python feature — no web UI, no shadcn.
> All "styling" is ANSI escape sequences and character selection.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

Terminal spacing is measured in character columns and rows, not pixels.

| Token | Value | Usage |
|-------|-------|-------|
| indicator-margin-right | 1 column | Space between indicator text and rightmost terminal column |
| indicator-width | 3 columns | Reserved display width for all indicator states (see Frame Inventory) |
| prompt-safe-gap | 0 columns | Indicator does not add spacing relative to user prompt — cursor restore handles positioning |

**Web spacing scale (4/8/16/24/32/48/64px): not applicable.** This UI lives in terminal column/row space.

**Exceptions:** None.

## Typography

Terminal text is monospace by definition. "Typography" here means character choice, ANSI weight/color codes, and display width.

| Role | Characters | ANSI Style | Display Width |
|------|-----------|------------|---------------|
| Searching frame (emoji mode) | See Frame Inventory | plain (no style codes) | 2 columns each |
| Searching frame (ASCII fallback) | See Frame Inventory | `\033[36m` (cyan) | 1 column each |
| Alert frame (emoji mode) | `⚠️` alternating with `!!` | plain | 2 columns |
| Alert frame (ASCII fallback) | `(!)\033[0m` alternating with `   ` | `\033[1;33m` (bold yellow) | 3 columns |
| Hidden state | empty string | none | 0 columns |

**Note:** Rich is NOT used by the daemon. All styling is raw ANSI escape codes written to `/dev/tty` (Unix) or `sys.stderr` (Windows). This avoids any Rich Live context conflict.

---

## Color

Terminal color is declared as ANSI codes, not hex. Colors align with the existing `neon` theme from `src/devmon/render/themes.py`.

| Role | ANSI Code | Rich Equivalent | Usage |
|------|-----------|----------------|-------|
| Searching (ASCII) | `\033[36m` | `cyan` | ASCII fallback search frames |
| Alert | `\033[1;33m` | `bold yellow` | ASCII fallback alert state `(!)` |
| Reset | `\033[0m` | — | Always appended after styled text to prevent bleed |
| Emoji mode | no ANSI codes | — | Emoji carry their own visual weight; no color wrapping |

**60/30/10 rule is not applicable** to a 3-character terminal widget. Color usage is: searching=cyan (theme-aligned), alert=bold yellow (warning semantic), reset appended always.

**Destructive color:** Not applicable. Indicator has no destructive actions.

## Frame Inventory

The complete set of characters and sequences the executor must implement.

### Searching State — Emoji Mode (default when emoji supported)

A 4-frame walking animation cycling at 500ms intervals (one frame per loop tick).

| Frame Index | Characters | Notes |
|-------------|-----------|-------|
| 0 | `🐾 ` | paw print + trailing space (3 display columns total — 2 for emoji + 1 space) |
| 1 | ` 🐾` | leading space + paw print |
| 2 | `🐾 ` | same as frame 0 |
| 3 | `✨ ` | sparkle — "found something" moment in cycle |

**Source:** Claude's Discretion (CONTEXT.md) — exact emoji characters delegated to this spec.

**Total frame count:** 4. `frame_idx = (frame_idx + 1) % 4`

### Searching State — ASCII Fallback

| Frame Index | Characters (with ANSI) | Display Width |
|-------------|----------------------|---------------|
| 0 | `\033[36m.\033[0m  ` | 3 columns |
| 1 | `\033[36m..\033[0m ` | 3 columns |
| 2 | `\033[36m...\033[0m` | 3 columns |
| 3 | `\033[36m..\033[0m ` | 3 columns |

Mirrors the existing `_SEARCH_FRAMES = [".", "..", "..."]` pattern from `commands/prompt.py` (canonical ref). Cyan color aligns with `neon` theme border color.

### Alert State — Emoji Mode

A 2-frame flash at 500ms intervals.

| Frame Index | Characters | Notes |
|-------------|-----------|-------|
| 0 | `⚠️ ` | warning emoji + trailing space (3 display columns) |
| 1 | `!! ` | ASCII "!!" — alternates with emoji for blinking effect |

**Source:** D-04 (locked decision) — alert uses exclamation flash alternating with emoji.

### Alert State — ASCII Fallback

| Frame Index | Characters (with ANSI) | Display Width |
|-------------|----------------------|---------------|
| 0 | `\033[1;33m(!)\033[0m` | 3 columns |
| 1 | `   ` | 3 spaces (blank — creates blink effect) |

### Hidden State

| Condition | Output |
|-----------|--------|
| `indicator_hidden = True` in save file | Empty string — indicator clears itself and writes nothing |

**Clear sequence:** `CURSOR_SAVE + move_to_col(col) + "   " + CURSOR_RESTORE` — overwrite with 3 spaces, then hide.

## ANSI Positioning Contract

All writes follow this exact sequence. Executor must not deviate.

```
\033[s                    # SCO cursor save (save current cursor position)
\033[{col}G               # Move to absolute column col on current line
{indicator_text}          # Frame content (3 display columns, including ANSI codes)
\033[u                    # SCO cursor restore (return to saved position)
```

**Column calculation:**
```
col = max(1, terminal_cols - indicator_display_width - indicator_margin_right)
    = max(1, terminal_cols - 3 - 1)
    = max(1, terminal_cols - 4)
```

Where `terminal_cols = shutil.get_terminal_size(fallback=(80, 24)).columns`.

**Source:** D-06 (locked) — ANSI cursor positioning. RESEARCH.md Pattern 3. Use SCO form `\033[s`/`\033[u` (not DEC `\033 7`/`\033 8`) per research recommendation.

**Write target:**
- Unix: `/dev/tty` (direct terminal device, bypasses pipes)
- Windows: `sys.stderr` (usually a tty on Windows; `CONOUT$` via ctypes is optional upgrade)

**On `OSError`:** Silently skip the write. Never crash or print an error.

## State Transition Contract

| Game State | Indicator Behavior | Detection |
|------------|--------------------|-----------|
| No encounter queued | Searching animation (cycling frames) | `state.encounter_queue is None` |
| Encounter queued | Alert animation (flashing frames) | `state.encounter_queue is not None` |
| Battle active | Hidden — clears indicator, writes nothing | `state.indicator_hidden is True` |
| Save file unreadable | Searching animation (fail-safe default) | `json.loads()` raises any exception |
| No save file exists | Searching animation (fail-safe default) | `FileNotFoundError` |

**State read frequency:** Once per 500ms animation loop tick. Read only `encounter_queue` and `indicator_hidden` keys from raw JSON — do not instantiate full Pydantic model. Keep read under 10ms.

**Battle detection:** Via `indicator_hidden: bool = False` field added to `GameState` (Option B from RESEARCH.md Pattern 4). Battle command sets `True` at start, `False` at end. Requires schema_version bump to 11.

**Source:** D-05 (locked) — daemon reads JSON save file directly. RESEARCH.md Pattern 4 recommendation.

## Emoji Support Detection

Determines whether emoji or ASCII fallback frames are used. Checked once at daemon startup; result cached for the process lifetime.

**Detection logic (in priority order):**

1. Check `TERM` environment variable — if `dumb` or empty: use ASCII fallback
2. Check `COLORTERM` — if `truecolor` or `24bit`: emoji likely supported
3. Check locale — if `UTF-8` in `LC_ALL` or `LANG`: emoji likely supported
4. Default: use ASCII fallback (safe over unknown)

**Explicit override:** Config key `ui.indicator_emoji = true/false` in `config.toml`. When set, skip detection and use configured value. Defaults to `true` on macOS/Linux, `false` on Windows.

**Source:** D-03 (locked) — accept emoji rendering variance, provide plain-text fallback. Exact detection method is Claude's Discretion.

## Copywriting Contract

This is a terminal animation — no headings, body text, or CTAs. Copywriting is limited to status and error messages from the `devmon indicator` CLI subcommand.

| Element | Copy |
|---------|------|
| Daemon already running | `Indicator already running (PID {pid})` |
| Daemon started | `Indicator started (PID {pid})` |
| Daemon stopped | `Indicator stopped` |
| Daemon not running (on stop) | `Indicator not running` |
| Status: running | `Indicator running (PID {pid}), state: {searching|alert|hidden}` |
| Status: stopped | `Indicator not running` |

**Empty state:** Not applicable — the daemon has no "empty state" UI surface.

**Error state:** Any `OSError` writing to `/dev/tty` is silently swallowed. The daemon never prints error messages to the terminal during normal operation (it would corrupt terminal output).

**Destructive actions:** None. `devmon indicator stop` terminates the daemon but requires no confirmation — it is non-destructive and immediately reversible by `devmon indicator start`.

## Terminal Compatibility Contract

| Terminal / Environment | Behavior |
|-----------------------|----------|
| xterm, iTerm2, Alacritty, kitty | Full emoji mode, ANSI cursor positioning |
| VS Code integrated terminal | Full emoji mode (TERM=xterm-256color) |
| tmux pane | Full ANSI cursor positioning; emoji depends on outer terminal |
| SSH session (remote) | ASCII fallback recommended — emoji width unreliable over SSH |
| Windows Terminal (wt.exe) | Full emoji mode; use `sys.stderr` write path |
| `TERM=dumb` | Indicator disabled entirely — write nothing, no ANSI |
| Narrow terminal (<20 cols) | Indicator disabled — `shutil.get_terminal_size()` returns width; if width < 20 skip write |
| Bash readline conflict | Known cosmetic issue — continuous writes appear between prompt and typed input. Accepted behavior per RESEARCH.md. Document in README. |

**Source:** D-06 (locked) — ANSI works across bash/zsh/fish. RESEARCH.md "Core technical challenge" section.
