<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify shell hooks work in a real terminal session</name>
  <what-built>
    devmon hook install/uninstall (bash/zsh), shell hook snippets, event log processing, devmon track test-pass
  </what-built>
  <how-to-verify>
    Complete these steps in a real bash or zsh terminal (not the Python test runner):

    **Step 1 — Install hooks:**
    ```
    devmon hook install
    ```
    Expected: "bash hook installed in ~/.bashrc" and "zsh hook installed in ~/.zshrc" (or "already installed" if run before)

    **Step 2 — Check rc files:**
    ```
    grep "devmon hook" ~/.bashrc
    grep "devmon hook" ~/.zshrc
    ```
    Expected: Each file contains `# --- devmon hook begin ---` and `# --- devmon hook end ---` blocks

    **Step 3 — Activate hooks in current session:**
    ```
    source ~/.bashrc    # (or source ~/.zshrc if using zsh)
    ```

    **Step 4 — Run a few commands:**
    ```
    ls
    pwd
    echo hello
    ```

    **Step 5 — Check event log has entries:**
    ```
    cat "$DEVMON_EVENT_LOG"
    ```
    (If DEVMON_EVENT_LOG is not set, check the platform default path shown by: `python -c "from devmon.config.defaults import DEFAULT_CONFIG; print(DEFAULT_CONFIG['shell']['event_log'])"`)
    Expected: JSON Lines entries like `{"ts":...,"exit":0,"dur":...,"cwd":"...","type":"cmd"}`

    **Step 6 — Process events with devmon:**
    ```
    devmon status
    ```
    Expected: devmon runs without error; XP in status output should reflect earned XP

    **Step 7 — Test explicit tracking:**
    ```
    devmon track test-pass
    ```
    Expected: "Test pass recorded!" message; event log contains a "test_pass" entry

    **Step 8 — Uninstall hooks:**
    ```
    devmon hook uninstall
    grep "devmon" ~/.bashrc | head -3
    ```
    Expected: No devmon lines remain in .bashrc

    **Step 9 — Verify reinstall is clean:**
    ```
    devmon hook install
    grep -c "devmon hook begin" ~/.bashrc
    ```
    Expected: Output is "1" (not "2" — idempotency confirmed)
  </how-to-verify>
  <resume-signal>
    Type "approved" if all 9 steps succeed.
    Or describe which step failed and what error was shown.
  </resume-signal>
</task>
