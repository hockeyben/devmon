Present the human verification checklist. Ensure a queued encounter exists before testing. If none exists, help the user trigger one or manually queue one in the save file.

Verification covers 6 test areas:
1. Battle screen rendering (BATL-05, UI-03)
2. Attack and turn order (BATL-01, BATL-02, BATL-03)
3. Capture flow (CAPT-01, CAPT-05)
4. Flee (D-03)
5. Empty state (D-06, CLI-02)
6. devmon --help command listing

**what-built:** Complete battle and capture system: turn-based battle with Rich Live rendering, capture shake animation, victory/defeat/capture/flee result screens, HP bar color transitions, action menu with all options.

**how-to-verify:**

**Setup:** Ensure you have a queued encounter. If none exists, run several shell commands to trigger one, or manually add an encounter to your save file.

**Test 1 -- Battle screen rendering (BATL-05, UI-03):**
1. Run `uv run devmon battle`
2. Verify: stacked panel layout with enemy creature on top, your creature on bottom
3. Verify: HP bars show green color with full health
4. Verify: ASCII art renders correctly for both creatures
5. Verify: Action menu shows all 6 options (Items grayed out)
6. Verify: Turn narration line shows "Turn 1 -- Battle begins!"

**Test 2 -- Attack and turn order (BATL-01, BATL-02, BATL-03):**
1. Select [1] Attack
2. Verify: faster creature acts first (check speed stats)
3. Verify: damage narration shows compact format with emoji
4. Verify: HP bars update to reflect damage dealt
5. Verify: HP bar color changes when crossing 50% and 25% thresholds
6. Attack several more times to see the full color transition

**Test 3 -- Capture flow (CAPT-01, CAPT-05):**
1. Weaken the wild creature to low HP (red bar)
2. Select [3] Capture
3. Verify: shake animation plays with pauses between lines
4. Verify: on success -- capture result screen with rewards
5. Verify: on failure -- "broke free!" in red, battle continues

**Test 4 -- Flee (D-03):**
1. Start a new battle (queue another encounter)
2. Select [6] Flee
3. Verify: flee message, immediate exit

**Test 5 -- Empty state (D-06, CLI-02):**
1. Run `uv run devmon battle` with no encounter queued
2. Verify: friendly empty state message

**Test 6 -- devmon --help:**
1. Run `uv run devmon --help`
2. Verify: "battle" appears in command list
