Create `src/devmon/commands/battle.py` as a Typer subcommand. This is the CLI layer — allowed to import from engine/, render/, persistence/, and models/.

```python
"""devmon battle — engage the queued wild creature encounter.

CLI layer: orchestrates battle loop using engine/ for logic, render/ for display,
and persistence/ for save/load. Must NOT be imported by domain modules.
"""
from __future__ import annotations
import typer

app = typer.Typer()
```

**battle_cmd() function** — the main entry point, registered via `@app.callback(invoke_without_command=True)`:

1. **Load state**: `state = load()`. If None or `state.encounter_queue is None`: print empty state message "No wild encounter queued. Keep coding \u2014 one will appear soon!" and exit 0. (D-06)

2. **Resolve party lead** (D-04, bootstrap per RESEARCH Pattern 5):
   - `_resolve_party_lead(state)` returns first non-fainted OwnedCreature from creature_collection
   - If creature_collection is empty: call `_bootstrap_starter(state)` to create a default Bugbyte at level 5, add to creature_collection and party
   - If all creatures fainted: print "All your creatures are fainted! They need rest before battle." and exit
   - Look up the CreatureTemplate for the active creature via `get_creature(owned.template_id)`

3. **Set up wild creature state** (RESEARCH Pitfall 7):
   - Create a `dataclass WildBattleState` with template, level, current_hp, max_hp, encounter_type, rarity
   - Compute max_hp from `compute_max_hp(wild_template, encounter_level)`
   - Set current_hp = max_hp

4. **Resolve player creature HP** (RESEARCH Pitfall 5):
   - If owned.current_hp is None: set it to `compute_max_hp(template, owned.level)`

5. **Battle loop** with Rich Live (D-19, UI-SPEC):
```python
from rich.live import Live
from rich.console import Console

console = Console()
battle_active = True
turn = 1
last_narration = "Battle begins!"

with Live(auto_refresh=False, console=console) as live:
    while battle_active:
        # Build renderable
        wild_panel = render_battle_creature_panel(wild_template, wild.current_hp, wild.max_hp, wild.level, "WILD", wild.rarity)
        player_panel = render_battle_creature_panel(player_template, player_owned.current_hp, player_max_hp, player_owned.level, "YOUR", player_template.rarity)
        
        abilities = get_available_abilities(player_template.abilities, player_owned.level)
        ability_name = abilities[-1].name if abilities else None
        can_switch = len([c for c in state.creature_collection if not c.is_fainted and c.template_id != player_owned.template_id]) > 0
        
        menu = render_action_menu(ability_name, can_switch, turn)
        renderable = build_battle_renderable(wild_panel, player_panel, turn, last_narration, menu)
        live.update(renderable)
        live.refresh()
        
        choice = input("  Enter choice [1-6]: ").strip()
        # Process choice...
```

6. **Action handling inside loop**:

   **[1] Attack** (BATL-02):
   - Determine turn order via `determine_turn_order(player_speed, wild_speed)` (D-01)
   - For the faster creature:
     - Compute type_effectiveness, roll_crit, compute_damage
     - Apply damage to target HP
     - Build narration: "{name} \u2694\ufe0f {dmg} dmg" + "(SE!)" or "(NVE)" or "(CRIT!)" suffixes
     - Check if target fainted
   - Then the slower creature acts (if not fainted)
   - Increment turn

   **[2] Special Ability** (BATL-02, D-10):
   - If no abilities available: print "This creature hasn't learned a special ability yet." and re-prompt
   - Otherwise: use the highest-level available ability
   - Same damage flow as Attack but multiply by ability.damage_multiplier and use ability.type for effectiveness
   - Narration: "{name} \U0001f525 {dmg} dmg" + type suffixes

   **[3] Capture** (CAPT-01, CAPT-07):
   - Exit Live context (critical — UI-SPEC)
   - Compute hp_percent = wild.current_hp / wild.max_hp
   - Compute capture_chance via compute_capture_chance(base_rate, hp_percent, 1.0) (basic capsule for MVP)
   - Roll capture via attempt_capture(capture_chance)
   - Run capture animation via run_capture_animation(console, "Basic Capsule", wild_name, wild_rarity, success)
   - If success (CAPT-05): create OwnedCreature from wild, add to creature_collection, increment total_creatures_captured, compute capture rewards, save state, render capture screen, break
   - If fail (CAPT-06): check resolve_wild_flee_after_failed_capture() — if wild flees, render_wild_fled_message, save, break. Otherwise wild gets a turn, re-enter Live.

   **[4] Switch Creature** (BATL-08):
   - If only one live party member: print "No other party members available." and re-prompt
   - Show list of non-fainted creatures (excluding current active): numbered, with name, level, HP
   - Get choice, switch active creature (costs a turn — wild acts after)
   - Narration: "{new_creature_name} switched in!"

   **[5] Items**: Print "Items not available yet. Coming in a future update." and re-prompt.

   **[6] Flee** (D-03):
   - Clear encounter_queue, increment flee_count, save state
   - Exit Live, render_flee_message, break

   **Invalid input**: Print "Invalid choice. Enter 1, 2, 3, 4, 5, or 6." and re-prompt.

7. **Wild creature turn** (after player action):
   - wild_creature_ai determines action
   - Compute damage and apply to player creature
   - If player creature faints (BATL-07): apply_faint, check for other live party members
     - If others exist (BATL-08): auto-switch to next non-fainted creature (or show choice if multiple)
     - If none exist (D-05): save state, exit Live, render_defeat_screen, break
   - Build wild narration line

8. **Victory flow** (BATL-06):
   - When wild creature HP <= 0: compute_battle_rewards, apply to player (add XP, currency), apply_creature_xp to active creature, increment battles_won, save state, exit Live, render_victory_screen

9. **State save**: ALWAYS call `save(state)` BEFORE rendering any result screen (Pitfall 4).

10. **Healing**: After battle resolution (any outcome), auto-heal all party creatures to full HP. This is the MVP healing mechanism (Claude's discretion). Set `current_hp = None` (which means full HP) and `is_fainted = False` for all owned creatures. Save again.

**Helper functions**:
- `_resolve_party_lead(state) -> OwnedCreature | None`: first non-fainted creature in creature_collection
- `_bootstrap_starter(state) -> OwnedCreature`: creates OwnedCreature(template_id="bugbyte", level=5), appends to creature_collection and party
- `_get_party_creatures(state) -> list[OwnedCreature]`: returns creatures from creature_collection whose template_id is in state.party (or all if party is empty for bootstrap)

Convert remaining xfail stubs (BATL-01, BATL-02, BATL-08, CAPT-01, CAPT-05, CAPT-07) to real tests using Typer's CliRunner with simulated input.
