# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Battle and Capture.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase introduces the battle screen as the primary new UI surface — the
> most complex Rich layout in the project so far. Six distinct UI surfaces:
> (1) battle screen (stacked panel layout with both creatures), (2) action menu
> with turn prompt, (3) per-turn narration one-liner, (4) capture shake animation
> sequence, (5) battle result screen (win/lose/flee/capture), and (6) the "no
> encounter queued" empty state for `devmon battle`.
> All rendering uses Rich — no web, no GUI, no shadcn. Spacing and typography
> sections map to Rich layout concepts, not CSS.

## Spacing Scale

Rich panels and terminal layouts use character-cell units, not CSS pixels.
The 8-point pixel scale is mapped to character equivalents below.
This table is identical to Phase 5 — no new exceptions in Phase 6.

| Token | Pixel equiv | Char equiv | Usage |
|-------|-------------|------------|-------|
| xs | 4px | 1 char | Inline padding inside Text blocks |
| sm | 8px | 2 chars | Panel padding (Rich default: 1 char each side) |
| md | 16px | 4 chars | Gap between stacked creature panels |
| lg | 24px | 6 chars | Blank lines between battle screen sections |
| xl | 32px | 8 chars | Not used in Phase 6 |
| 2xl | 48px | 12 chars | Not used in Phase 6 |
| 3xl | 64px | 16 chars | Not used in Phase 6 |

Exceptions:
- HP bar width: 20 characters fixed (maps to md token, fits cleanly in panel).
- ASCII art line width: max 40 chars per line (inherited from Phase 4 constraint).
- Action menu items: 2-space indent, matching Phase 5 encounter menu convention.
- Touch targets: not applicable (terminal, keyboard-only).

Source: 05-UI-SPEC.md Spacing Scale.

## Typography

Terminal rendering has no font size — all text is monospace at the terminal's
native size. "Size" below maps to Rich semantic styling (bold, dim, etc.).

| Role | Rich Style | Usage |
|------|-----------|-------|
| Display | `bold {rarity_color}` | Creature names in battle panel titles |
| Heading | `bold white` | Action menu header ("Your turn! What will you do?"), battle result heading |
| Body | `white` | Action menu items, narration one-liners, turn result lines |
| Label | `dim {theme.stat_key}` | HP label in battle panel, stat key labels |
| Muted | `dim white` | Panel subtitles (rarity + type), Items grayed-out action |
| HP numeric | `{hp_color}` (green/yellow/red) | Current HP numeric value alongside bar — color matches bar |

## Color

This project uses Rich style strings, not hex values. The 60/30/10 split maps
to semantic layer responsibilities in the terminal.

| Role | Rich Style | Usage |
|------|------------|-------|
| Dominant (60%) | terminal background (no fill) | Battle screen interior, narration text background |
| Secondary (30%) | `dim` overlays, panel borders via `theme["border"]` | Panel borders, stat label dimming, grayed-out Items action |
| Accent (10%) | rarity color per RARITY_COLORS map (see below) | Both creature panel borders + names, capture success banner |
| Destructive | `bold red` | Wild creature faint message ("Bugbyte fainted!"), battle-lost result heading |

Accent reserved for: enemy creature panel border and name, player creature panel
border and name, capture success banner. Never used for narration text, HP
numeric values, action menu items, or the empty state message.

Source: CONTEXT.md D-16, D-17, themes.py RARITY_COLORS (verified in codebase).

### HP Bar Color Thresholds

Both the ASCII bar fill characters and the numeric HP value use the same color.
Color changes are applied at render time based on `(current_hp / max_hp)`.

| Threshold | Rich Style | Condition |
|-----------|-----------|-----------|
| Healthy | `green` | hp_percent > 0.50 |
| Caution | `yellow` | 0.25 < hp_percent <= 0.50 |
| Critical | `red` | hp_percent <= 0.25 |

HP bar format: `[{color}]{"█" * filled}{"░" * empty}[/{color}] {current}/{max}`
Bar width: 20 characters fixed. Filled count = `round(hp_percent * 20)`.

Source: CONTEXT.md D-17.

## UI Surface Contracts

This section defines the six distinct UI surfaces introduced in Phase 6.

### Surface 1: Battle Screen (Full Turn Redraw)

Triggered: `devmon battle` when an encounter is queued. Redrawn each turn via
Rich `Live` layout. Old turn state is cleared on each redraw — no scrollback
history during battle.

Layout (top to bottom, single column, `expand=False` for each panel):

```
┌─────────────────────────────────────────┐
│  WILD: {creature_name}   [{rarity}]     │  ← enemy creature panel (rarity border)
│  {ascii_art}                            │
│  HP ████████████░░░░░░░░ 45/80          │
│  LVL {level}  Type {type}               │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  YOUR: {creature_name}   [{rarity}]     │  ← player creature panel (rarity border)
│  {ascii_art}                            │
│  HP ████████████████████ 80/80          │
│  LVL {level}  Type {type}               │
└─────────────────────────────────────────┘

  Turn {N} — {last_action_narration_line}

  Your turn! What will you do?

  [1] Attack
  [2] Special Ability  ({ability_name})
  [3] Capture
  [4] Switch Creature
  [5] Items  (coming soon)
  [6] Flee

  Enter choice [1-6]:
```

Rules:
- Two Rich panels stacked vertically (enemy on top, player on bottom). Matches
  classic creature-battle layout from CONTEXT.md D-16.
- Both panels use `box.ROUNDED`, `expand=False`, border color = creature rarity
  color (RARITY_COLORS lookup for each creature independently).
- Enemy creature panel title: `"WILD: {creature_name}"` styled
  `bold {rarity_color}`. Player creature panel title:
  `"YOUR: {creature_name}"` styled `bold {rarity_color}`.
- HP bar rendered inline in the panel body immediately below the ASCII art.
  Format: `HP ` (dim stat_key) + `{bar}` + ` {current}/{max}` — bar color and
  numeric value both use the HP threshold color (see Color section).
- ASCII art: uses `template.ascii_art` lines joined with newlines, styled
  `template.primary_color` (same as `render_creature_panel()`).
- Stat row below HP bar: `LVL {level}  Type {type}` — stat_key / stat_value
  Rich styles. Single line, no second column.
- Turn narration line (between panels and action menu): `"Turn {N} — {last_action}"`.
  `Turn {N}` in `dim white`. Narration segment in `white`. Single line.
  On Turn 1 (no previous action yet): `"Turn 1 — Battle begins!"` in `dim white`.
- Action menu heading: `"  Your turn! What will you do?"` — 2-space indent,
  `bold white`. Blank line above and below heading.
- Menu items: 2-space indent, `white`, one per line:
  - `[1] Attack` — always active
  - `[2] Special Ability  ({ability_name})` — ability name in `dim white`
    parenthetical. If no ability learned yet: `[2] Special Ability  (none yet)`
    in full `dim white` (unselectable — shows message if chosen).
  - `[3] Capture` — always active
  - `[4] Switch Creature` — grayed (`dim white`) if only one party member alive
  - `[5] Items  (coming soon)` — full `dim white` (non-functional until Phase 8)
  - `[6] Flee` — always active, always succeeds (CONTEXT.md D-03)
- Blank line between last menu item and input prompt.
- Input prompt: `"  Enter choice [1-6]: "` — 2-space indent, `white`.
  Use `typer.prompt()` or `input()` — NOT Rich Prompt.
- Full screen is cleared and redrawn each turn. Use Rich `Live` with
  `auto_refresh=False`, call `live.update(renderable)` + `live.refresh()` per
  turn. This discards turn history but produces a clean readable layout per D-19.

Source: CONTEXT.md D-16, D-17, D-18, D-19, D-02, D-03.

### Surface 2: Per-Turn Narration One-Liner

Visible: appears in the "Turn {N} — {narration}" line of the battle screen,
updated each full-screen redraw. Not a separate surface — embedded in Surface 1.

Format patterns (compact/emoji-based per D-18):

| Event | Format |
|-------|--------|
| Player normal attack | `{player_name} ⚔️ {damage} dmg` |
| Player attack + super effective | `{player_name} ⚔️ {damage} dmg (SE!)` |
| Player attack + not very effective | `{player_name} ⚔️ {damage} dmg (NVE)` |
| Player attack + critical hit | `{player_name} ⚔️ {damage} dmg (CRIT!)` |
| Wild creature normal attack | `{wild_name} ⚔️ {damage} dmg` |
| Wild creature special ability | `{wild_name} 🔥 {damage} dmg` |
| Player uses special ability | `{player_name} 🔥 {damage} dmg` |
| Player switches creature | `{player_name} switched in!` |
| Capture attempt (turn cost) | `You threw a {item_name}!` |
| Creature fainted (player) | `{player_name} fainted!` in `bold red` |
| Creature fainted (wild) | `{wild_name} fainted!` in `bold red` |
| Battle begins | `Battle begins!` in `dim white` |

Rules:
- Maximum one narration line displayed at a time (current turn's outcome only).
- Compact format — never verbose prose. "⚔️ 12 dmg" not "lunges forward with a
  powerful attack."
- Type effectiveness suffix: `(SE!)` = super effective, `(NVE)` = not very
  effective. No suffix = neutral. Applied after damage value.
- `CRIT!` suffix takes precedence over type effectiveness suffix when both apply:
  `⚔️ 24 dmg (CRIT! SE!)`.
- Emoji selection: ⚔️ for normal physical attacks, 🔥 for special/elemental
  abilities. No per-type emoji differentiation for MVP.

Source: CONTEXT.md D-18.

### Surface 3: Capture Shake Animation Sequence

Triggered: Player selects [3] Capture during their turn. Replaces the normal
turn redraw with a sequential text animation before resolving.

Sequence (printed line by line with brief pause between each line):

```
You threw a {item_name}!

  The capsule wobbles...
  It shakes again...
  One more shake...
```

Then one of two outcomes:

**Success:**
```
  ✨ CLICK! {creature_name} was captured! ✨
```
Styled: `bold {rarity_color}` for the entire success line.

**Failure:**
```
  {creature_name} broke free!
```
Styled: `bold red` for the entire failure line.

Rules:
- Animation is sequential `console.print()` calls with `time.sleep(0.6)` between
  "wobbles" lines to build tension (D-12).
- Number of shake lines displayed: 3 lines always (regardless of outcome — result
  revealed only after all shakes). Provides consistent tension.
- On success: print success line, then print capture reward one-liner (see
  Surface 5), then exit battle (encounter cleared).
- On failure: print failure line, then resume normal battle turn order — wild
  creature gets their action next (they may flee per D-14).
- Rich `Live` is paused during the animation sequence (exit `Live` context,
  print animation lines, re-enter `Live` for next turn if battle continues).
- Item name in "You threw a {item_name}!": use item's display name
  (e.g., "Basic Capsule", "Great Capsule", "Ultra Capsule", "Master Capsule").
  For Phase 6 MVP where items system is not yet built: use "Basic Capsule"
  as the default item name always.

Source: CONTEXT.md D-12, D-14.

### Surface 4: Battle Result Screen

Triggered: Battle ends (player wins by defeating wild creature, party wipe,
capture success, or flee). Replaces battle screen entirely.

**Win (wild creature defeated):**
```
┌─────────────────────────────────────────┐
│  Victory!                               │
└─────────────────────────────────────────┘

  {player_creature_name} defeated {wild_name}!

  Rewards:
    Player XP:   +{xp_amount}
    {creature}:  +{xp_amount} XP
    Currency:    +{currency_amount} bits

  Press Enter to continue.
```
Panel border: `bold yellow` (victory accent — independent of rarity).
Panel title: `"Victory!"` in `bold yellow`.
Reward lines: label in `dim white`, value in `bold white`.

**Capture success (shown via capture animation then this result):**
```
┌─────────────────────────────────────────┐
│  {creature_name} Captured!              │
└─────────────────────────────────────────┘

  {creature_name} was added to your collection.

  Rewards:
    Capture Bonus XP: +{xp_amount}
    Currency:         +{currency_amount} bits

  Press Enter to continue.
```
Panel border: rarity color of the captured creature.
Panel title: `"{creature_name} Captured!"` in `bold {rarity_color}`.

**Defeat (all party fainted):**
```
┌─────────────────────────────────────────┐
│  Defeated...                            │
└─────────────────────────────────────────┘

  Your party was wiped out. No rewards this time.
  Your creatures need rest before their next battle.

  Press Enter to continue.
```
Panel border: `dim red`.
Panel title: `"Defeated..."` in `bold red`.
Body text: `dim white` (low-drama, non-punishing per D-05).

**Flee:**
```
You fled from {wild_name}. Encounter lost.
```
Single `console.print()` line — no panel. `{wild_name}` styled `bold {rarity_color}`.
No "press Enter" prompt — exits immediately.

Rules:
- Win and Capture results use `box.ROUNDED`, `expand=False`.
- Defeat result uses `box.ROUNDED`, `expand=False`.
- All panels: blank line above and below.
- "Press Enter to continue." prompt: `input("")` (empty prompt, just waits for
  Enter keypress). Returns to shell after dismissal.
- No currency display on Defeat result (no rewards earned — D-05).

Source: CONTEXT.md D-05, D-06, BATL-06, BATL-07, CAPT-05.

### Surface 5: Creature Faint / Switch Notification (In-Battle)

Triggered: A battle creature (player's or wild's) reaches 0 HP. Displayed in
the turn narration line before the next full screen redraw.

When player's active creature faints (and party has another live member):
```
{player_name} fainted! Switching to next party member...
```
Line displayed for one turn, then battle screen redraws with next party creature.

When player's active creature faints (no remaining party members):
→ immediately trigger Defeat result screen (Surface 4).

When wild creature faints:
→ immediately trigger Win result screen (Surface 4).

Rules:
- Faint message: `bold red`. Single line in narration position.
- "Switching to next party member..." in `dim white` on same line after faint
  message (or second line if line length is a concern — prefer single line).
- Auto-switch to next non-fainted party member (no player choice prompt needed
  when only one option remains; show prompt when multiple live members exist per
  BATL-08).

Source: CONTEXT.md D-04, BATL-07, BATL-08.

### Surface 6: Empty State for `devmon battle` (No Encounter Queued)

Triggered: `devmon battle` when `GameState.encounter_queue` is `None`.

Format: Single `console.print()` call — no panel, no border.

```
No wild encounter queued. Keep coding — one will appear soon!
```

Rules:
- Plain `white` text. No rarity color (no creature).
- No panel wrapper. Inline text only.
- Exactly one line. No follow-up prompt.
- Command exits cleanly after printing (exit code 0).

Source: CONTEXT.md D-06 (friendly message when no encounter queued).

## Interaction Flow Contract

### `devmon battle` — Happy Path (Encounter Queued, Player Wins)

```
User runs: devmon battle
  → load GameState
  → if encounter_queue is None: print empty state, exit 0
  → else:
      resolve party lead creature (Phase 6 bootstrap: first non-fainted owned creature)
      enter battle loop (Rich Live):
        each turn:
          redraw battle screen (enemy panel + player panel + narration + action menu)
          choice = input("  Enter choice [1-6]: ")
          if choice == "1":  Attack
              compute damage, apply to wild HP
              check wild HP → if 0: show Win result, exit
              wild creature takes their turn (AI: random ability or attack)
              check player HP → if 0: faint / switch / defeat flow
              next turn
          if choice == "2":  Special Ability
              if no ability: print message, re-prompt
              else: compute ability damage, apply, rest same as Attack
          if choice == "3":  Capture
              exit Live, run capture shake animation
              compute capture_chance = base_rate * (1 / hp_percent)  ← D-11
              if success: show Capture result, award XP + currency, exit
              if fail: print "broke free!", resume battle (wild gets turn, may flee)
          if choice == "4":  Switch Creature
              if only one live party member: print unavailable message, re-prompt
              else: show party list, player picks, switch active creature (costs turn)
          if choice == "5":  Items
              print "Items not available yet.", re-prompt
          if choice == "6":  Flee
              clear encounter_queue, save state
              print flee one-liner, exit 0
          else:
              print "Invalid choice. Enter 1, 2, 3, 4, 5, or 6."
              re-prompt
```

### Wild Creature AI

For Phase 6 MVP, wild creature AI is deterministic-random:
- Each turn: random choice between available actions (normal attack, or
  special ability if one exists for the creature at its level).
- No smart targeting, no defensive strategy.
- After a failed capture attempt: small chance (15%) to flee before taking
  their normal turn. If fled: print `"{wild_name} fled!"`, clear encounter_queue,
  save state, end battle (no reward — encounter lost).

Source: CONTEXT.md Claude's Discretion (wild creature AI behavior, flee chance
after failed capture).

### Creature Switch Flow (BATL-08)

Triggered when player selects [4] and multiple live party members exist, or
when active creature faints with multiple live members remaining.

```
  Choose your next creature:
  [1] {name_1}  Lv.{level}  HP {current}/{max}
  [2] {name_2}  Lv.{level}  HP {current}/{max}  (fainted)  ← dim red, unselectable
  ...
  Enter choice [1-{N}]:
```

Rules:
- Fainted creatures shown with `(fainted)` suffix in `dim red`. If selected,
  print "That creature can't battle right now." and re-prompt.
- Switching costs a turn (wild creature acts after switch resolves).
- Current active creature is not shown in the list (already in battle).

Source: CONTEXT.md D-04, BATL-08.

## Battle Screen Render Notes

### Rich Live Usage

The battle screen uses Rich `Live` for full-screen redraw per turn (D-19):

```python
from rich.live import Live

with Live(auto_refresh=False) as live:
    while battle_active:
        renderable = build_battle_renderable(state)
        live.update(renderable)
        live.refresh()
        choice = input("  Enter choice [1-6]: ")
        # process choice, update state
```

`build_battle_renderable()` returns a Rich `Group` containing:
1. Enemy creature panel (Panel with rarity border)
2. Blank line (Text(""))
3. Player creature panel (Panel with rarity border)
4. Blank line
5. Turn narration line (Text)
6. Blank line
7. Action menu lines (Text, assembled as one block)

Exit `Live` context before running the capture animation (Surface 3) and before
displaying any result screen (Surface 4).

### HP Bar Render Function

```python
def render_hp_bar(current: int, max_hp: int, width: int = 20) -> Text:
    """Returns Rich Text: colored bar + numeric value."""
    hp_percent = current / max_hp if max_hp > 0 else 0
    filled = round(hp_percent * width)
    empty = width - filled
    color = "green" if hp_percent > 0.50 else ("yellow" if hp_percent > 0.25 else "red")
    bar = Text()
    bar.append("HP ", style="dim cyan")           # stat_key equivalent
    bar.append("█" * filled, style=color)
    bar.append("░" * empty, style="dim white")
    bar.append(f" {current}/{max_hp}", style=color)
    return bar
```

This function lives in `render/battle.py` (new module for Phase 6).
It must not be inlined into the battle command.

Source: CONTEXT.md D-17.

### Creature Panel in Battle vs. Encounter

`render_creature_panel()` from `render/creatures.py` was designed for encounter
inspection — it shows all base stats and flavor text. The battle screen requires
a compact battle-panel variant showing only HP bar, level, and type (not full
stat block or flavor text).

Do NOT call `render_creature_panel()` directly for the battle screen. Instead,
create `render_battle_creature_panel()` in `render/battle.py`:

```python
def render_battle_creature_panel(
    template: CreatureTemplate,
    current_hp: int,
    max_hp: int,
    level: int,
    prefix: str,   # "WILD" or "YOUR"
    console: Console,
    theme: dict[str, str] | None = None,
) -> Panel:
    """Compact battle panel: ASCII art + HP bar + LVL/Type row."""
```

Body contains:
1. ASCII art (template.primary_color, same as render_creature_panel)
2. Blank line
3. HP bar (render_hp_bar output)
4. LVL/Type row: `LVL {level}  Type {type}`

No flavor text. No ATK/DEF/SPD stat block. No capture rate. Battle panels are
compact — full stats are for the encounter inspection screen (Phase 5).

Source: CONTEXT.md D-16, render/creatures.py (architecture pattern).
